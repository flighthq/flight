# path-boolean — Status

Directed 2026-07-09 for a full **AAA build, phased** (see charter Decisions). Correctness-first: each phase must pass its degeneracy-test bar before the next.

## Phase plan

- **Phase A — kernel + boolean ops (foundation).** Package scaffold + registration; the `PathBooleanBackend` seam (`setPathBooleanBackend`/`getPathBooleanBackend`/`createDefaultPathBooleanBackend`); the Martinez–Rueda default kernel; `unionPaths`/`intersectPaths`/`differencePaths`/`xorPaths` + `booleanPaths`; both fill rules (even-odd, non-zero); flatten-then-boolean over `flattenPath` with a tolerance. **Degeneracy tests:** disjoint, contained, overlapping, coincident edges, shared vertices, holes, self-intersecting input, empty/degenerate. This is the load-bearing pass.
- **Phase B — offsetting.** `offsetPath(path, delta, options)` with join types (miter + miterLimit / round / square / bevel) and end types (butt / round / square) for open paths; inflate and deflate. Tests: convex/concave joins, the miter-limit fallback, round-join arc tessellation, open vs closed, negative delta.
- **Phase C — simplify + clean.** `simplifyPath` (resolve self-intersections under a fill rule via the kernel) and `cleanPath` (epsilon-dedup coincident/near-collinear vertices). Tests: self-touching, spikes, duplicate points, collinear runs.
- **Phase D — batch + hardening.** `unionPaths` over an array (multi-path union), remaining op ergonomics, and a hardening pass widening degeneracy coverage + fuzz-style random-polygon round-trip invariants (`A ∪ B == B ∪ A`, `A ∖ A == ∅`, double-offset stability).

Each phase: dispatched to a builder, reviewed for geometry correctness, gated (`npm run check` + package tests + build), committed.

**Phase B → Phase D carry-forward** (offset-specific hardening, honest limits from the build): (1) concave corners emit the offset-edge intersection point rather than the vertex-route, because the kernel lacks a **positive fill rule** (Clipper's offset-cleanup fill) — correct for convex-deflation and mild concavity, but a concave feature narrower than `2·delta` or heavy self-overlap can still leave a self-intersecting ring nonZero can't clean like positive-fill would. **A positive-fill capability in the kernel is the real fix** (a kernel/Phase-D item). (2) collapse detection is a global heuristic (winding-inversion OR area-not-reduced for `delta<0`) — catches full collapse, misses partial collapse. (5) round-arc extreme is only a tessellation vertex within `arcTolerance` (exact-bounds consumers beware). (6) `delta===0` not special-cased (flows through as zero-width).

**Phase C → Phase D carry-forward / honest limits:**
- **The bowtie does NOT discriminate fill rules.** A figure-eight quad's two lobes have opposite winding, so both are nonzero — nonZero and evenOdd both yield the two triangles (area 1 each). The task brief expected bowtie to be the evenOdd/nonZero discriminator; it is not. The real discriminators (used in tests) are **self-overlap with the same winding**: two overlapping same-wound squares (nonZero = 1 ring area 28 union; evenOdd = 2 rings total 24, corner punched) and the **pentagram** (nonZero = 1 solid ring area ≈112.257; evenOdd = 3 rings, hollow center, total ≈77.568). Phase D should correct the brief's mental model and add a same-winding self-overlap invariant.
- `simplifyPath` empty/degenerate input → clean empty path (0 commands) via `resolvePathRegions`' `rings.length === 0` guard AND the kernel dropping zero-area contours (a collinear "triangle" also yields empty). This is a clean sentinel.
- `cleanPath` collinearity/coincidence use **absolute** `toleranceSq` comparisons (`perpDistSq ≤ tol²`, `dist²  ≤ tol²`) — same magnitude-relative caveat as the kernel/offset epsilons. Removal boundary is inclusive (`≤`), so a vertex exactly at tolerance is removed; "just over" is kept (tested).
- `cleanPath` spike detection is subsumed by the collinear test: a spike tip whose neighbors coincide hits the `lengthSq(P,S) ≤ tol²` degenerate-segment branch (→ remove); a colinear out-and-back spike hits `perpDistSq ≤ tol²`. A NON-colinear spike (tip off to the side, distinct non-coincident neighbors) is a genuine feature, not noise, and is correctly kept — flagged in case Phase D wants an angle-based needle removal.
- `cleanPath` closed/open inferred from first/last-vertex coincidence (within tol) — same open-polyline-with-coincident-endpoints misclassification caveat as offset.
- The forward-stack + wrap-seam cleanup is single-directional; pathological inputs where a seam removal re-exposes an interior collinearity many vertices away could in principle need another interior pass. Not observed on tested inputs; Phase D fuzz should probe it.

This dissolves the negatively-wound self-overlap the concave inner-miter emission produces on a feature narrower than 2·|delta| — the case nonZero kept as a spur. New test: a width-1 slot inflated by 1 (slot < 2·delta) closes to a single self-intersection-free ring (validated by simplify-invariance).
2. **Magnitude-relative epsilons.** Kernel `VERTEX_SNAP` (was absolute 1e-7) → `vertexSnap = extent · 1e-9` derived from the input bounding-box extent at the top of `computeMartinezBoolean` (module-scoped like `nextEventId`, single-entrant, no import side effect); `VERTEX_SNAP_FALLBACK` 1e-9 for zero extent. `INTERSECTION_EPS` was already relative (scaled by squared edge lengths in each comparison) — left as-is with a corrected comment. 3. **Flagged degeneracies.** cleanPath non-collinear spike: **decided keep** (a real protrusion, not vertex noise; angle-based needle removal is a separate simplify concern out of cleanPath's dedup scope) + test in `@flighthq/path`. compareSegments total-order stability and the classification perpendicular-offset sample (len·1e-4, already scale-relative) are exercised across 40 random self-overlapping polygons each by the fuzz harness — left as-is (exact-arithmetic comparator is a total order by construction). Ray cast through a strict x-extremum: defensively handled by `windingAt`'s half-open x-interval and made a measure-zero event by the midpoint-perpendicular sampling (samples never land exactly on a vertex x for tested inputs) — noted, not churned. **Genuine remaining limitations (honest):** (a) offset closed-vs-open inference still keys on first/last-vertex coincidence — an open polyline with coincident endpoints misclassifies, and this is NOT fixable from flattened geometry alone (closedness is lost in `flattenPath`; a real fix needs flatten to carry per-subpath closedness). (b) **Deflating a densely-tessellated arc is unstable** — inner-miter intersection of near-collinear edges shorter than |delta| loses area badly (observed ~40-94% on round-join double-offset). This is why the double-offset fuzz invariant uses a miter join (clean convex polygon, edges ≫ |delta|), where recovery is within 3%. Deflation of round-join output is a known weak spot; a robust fix wants edge-length-aware handling or the vertex-route restricted to genuine reflex corners.
