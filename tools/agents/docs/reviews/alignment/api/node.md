# API Alignment: @flighthq/node

**Verdict:** Naming is exemplary — every export spells out the full type word (`getNodeWorldTransformMatrix`, `computeViewportFillScale`) and allocation/teardown verbs are correct — but the package has a cluster of low-severity symmetry defects: getter parameters drift between `source` and `target`, the 2D/3D point-convert siblings disagree on `Readonly`/parameter names, `computeViewportRenderTransform` calls a `Viewport` `scene`, and one revision getter is missing `Readonly`.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `computeViewportRenderTransform` | Second parameter is named `scene` but typed `Viewport<Traits>`. `scene` is a different graph concept in the SDK (`@flighthq/scene`, `SceneNode`); naming a `Viewport` `scene` is actively misleading in a package that also exports `createViewport`. The `*Align`/`*Scale` siblings have no such object param to contrast with, so the mismatch is easy to miss. | Rename the parameter to `viewport` to match its type and the rest of the viewport family. |
| Low | `getNodeLocalBoundsRectangle`, `getNodeParentBoundsRectangle`, `getNodeWorldBoundsRectangle`, `getNodeLocalTransformMatrix`, `getNodeWorldTransformMatrix`, `getNodeWorldTransformMatrix4` | These pure read accessors name their parameter `target`, while the other read accessors in the package name it `source` (`getNodeChildAt`, `getNodeParent`, `getNodeRoot`, `getNodeAppearanceRevision`, `getNodeWidth`, `getNodeRuntime`). `target` is the package's mutation/out convention (`addNodeChild(target, …)`, `setNodeWidth(target, …)`, `invalidateNode(target)`), so using it on getters blurs the read/write signal. The cause is that these getters call an internal `ensure*` that lazily fills a cache — but that is an implementation detail, not a mutation the caller requested. | Rename the parameter to `source` on the bounds/transform getters for consistency with every other `get*` in the package and to reserve `target` for functions that mutate the node the caller passes. |
| Low | `getNodeWorldTransformRevision` | Takes `source: Node<Traits>` (mutable) while every sibling revision getter takes `Readonly<Node<Traits>>` (`getNodeAppearanceRevision`, `getNodeLocalBoundsRevision`, `getNodeLocalContentRevision`, `getNodeLocalTransformRevision`). It only reads `worldTransformId`, so the missing `Readonly` is an oversight. | Change to `source: Readonly<Node<Traits>>` to match the other four revision getters and the Readonly-by-default rule. |
| Low | `convertNodeVector2GlobalToLocal` / `convertNodeVector2LocalToGlobal` vs `convertNodeVector3GlobalToLocal` / `convertNodeVector3LocalToGlobal` | The 2D pair names the input `vector`; the 3D pair names the same role `point`. Same operation, sibling functions, divergent parameter vocabulary. | Pick one word for the input across both dimensions (`vector` reads best given the `Vector2Like`/`Vector3Like` types) so the 2D/3D convert family is symmetric. |
| Low | `convertNodeVector3GlobalToLocal` | Allocates a fresh `createMatrix4()` per call on a hot transform path, whereas the 2D twin `convertNodeVector2GlobalToLocal` allocates nothing (it uses `inverseMatrixTransformPointXY`). `boundsRectangle.ts` already demonstrates the pooled pattern (`acquireMatrix()`/`releaseMatrix()`); this function should not allocate per call. | Use `acquireMatrix4()`/`releaseMatrix4()` (or a module-scope scratch matrix like `_tempBoundsRectangle`) instead of `createMatrix4()` so the global→local convert is allocation-free like its 2D sibling. |
| Low | `getNodeSignals` | Pure read accessor returning `NodeSignals \| null`, but takes non-`Readonly` `source: Node<Traits>`. `getNodeRuntime` (also a getter) correctly takes `Readonly<Node<Traits>>`. (`enableNodeSignals` taking a mutable `source` is defensible since it lazily creates, but the plain getter should not.) | Change `getNodeSignals` to `source: Readonly<Node<Traits>>`. |
| Low | `createNode` (`createNodeRuntimeFactory` parameter) | The fourth parameter is named `createNodeRuntimeFactory` (stacking the `create` verb with the `Factory` noun) while the adjacent factory parameter is named `createData`. Inconsistent and slightly redundant — a factory is the thing, `create` is what it does. | Rename to `createRuntime` (mirrors `createData`) or `runtimeFactory` (mirrors the `NodeRuntimeFactory` type); drop the doubled `create…Factory`. |
| Low | `getEntityRuntime` casts (inherited) | `node.ts`, `boundsRectangle.ts`, `transform2d.ts`, and `transform3d.ts` all do `getEntityRuntime(target) as … & …Runtime` and then dereference cached fields with `!`. This rides on the upstream `@flighthq/entity` defect where `getEntityRuntime` is typed non-null but returns `undefined` for a runtime-less entity (see `api/entity.md`). The `!` non-null assertions here launder that expected-missing case. | Cross-package follow-up: once `getEntityRuntime` is typed honestly (`… \| null`) or an `ensureEntityRuntime` is added upstream, thread the nullability through these callsites instead of `as …!`. Surface to the user rather than fixing across the package boundary. |

## Clean

- **Full, unabbreviated type words everywhere.** Every export names its operand in full — `getNodeWorldTransformMatrix`, `getNodeLocalBoundsRectangle`, `computeViewportFillScale`, `swapNodeChildrenAt`, `invalidateNodeParentReference`. No `getNodeWTM`/`Bounds`/`Xform`-style shortening anywhere; `Matrix4` is kept distinct from `Matrix`.
- **Allocation discipline by verb.** Allocators are `create*` (`createNode`, `createNodeRuntime`, `createNodeSignals`, `createViewport`); the `compute*`/`ensure*`/`get*` bounds and transform functions write into an `out` parameter or a cached runtime field and reuse module-scope scratch (`_tempBoundsRectangle`, `_tempRectangle`). `boundsRectangle.ts` correctly brackets `acquireMatrix()`/`releaseMatrix()`. (The one exception — `convertNodeVector3GlobalToLocal` — is flagged above.)
- **`out`/`target` first-parameter convention is consistent.** Out-writing functions put the mutable destination first (`computeNodeBoundsRectangle(out, …)`, `computeViewportRenderTransform(out, …)`, `convertNodeVector2*(out, …)`); mutators put the node-to-mutate first (`addNodeChild(target, child)`, `setNodeWidth(target, value)`, `swapNodeChildren(target, child1, child2)`).
- **Alias-safe out-params.** `tryFastRecomputeWorldBoundsRectangle` reads the prior transform fields into locals (`_a…_ty`) before recomputing and comparing — the textbook read-before-write pattern. The `compute*`/`convert*` functions write `out` from distinct cached inputs (or read scalars first via `matrixTransformPointXY`), so `out === source` is safe.
- **Teardown verbs.** No teardown-verb misuse: there are no `dispose*`/`destroy*` exports (the package holds only GC-managed graph state and frees no native resources), and `acquire*`/`release*` appear only as correctly-bracketed pool calls inside `boundsRectangle.ts`, never leaked into the public API.
- **`Readonly<T>` on read paths.** Getters and predicates take `Readonly<Node<Traits>>` (`containsNodeChild`, `getNodeChildIndex`, `getNodeParent`, `getNodeRoot`, the revision getters) and second-position read inputs are `Readonly<Vector2Like>`/`Readonly<Vector3Like>`; `computeNodeWorldTransformRevision` correctly marks `parentRuntime` as `Readonly`. (The two omissions are the low-severity nits above.)
- **Accessor / boolean prefixes.** `get*` is used for all accessors and none return a boolean; the boolean-returning functions are correctly prefixed (`containsNodeChild`, `defaultNodeRuntimeCanAddChild`). No `get*`-returning-boolean and no unprefixed boolean getter.
- **Sentinels for expected-missing.** `getNodeChildAt`, `getNodeChildByName`, `removeNodeChildAt`, `getNodeParent`, `getNodeSignals` return `null` for not-found; `getNodeChildIndex` returns `-1`. `throw` is reserved for genuine API misuse (`addNodeChildAt`: null child, self-parenting, parent rejecting the child; `throwOutOfBoundsError` for an out-of-range index) — programmer errors, not expected failures.
- **Signal opt-in pattern.** `enableNodeSignals` is the owning-package `enable*` entry the map prescribes, returning a lazily-created `NodeSignals`; `getNodeSignals` returns `null` until enabled, so signal cost is opt-in and tree-shakable.
- **Header-layer types and import hygiene.** All cross-package types (`Node`, `NodeRuntime`, `Transform2DNode`, `Viewport`, `HasAppearance`, `Rectangle`, `Matrix4Like`, …) come from `@flighthq/types`; nothing cross-package is defined inline. Every source file keeps `import type {}` on its own line, separate from value imports (`createMatrix`, `EntityRuntimeKey`, `emitSignal`).
- **Thin barrel, no side effects.** `index.ts` is a flat re-export of the twelve modules, `"sideEffects": false`, single `.` export; no `registerRenderer`/listener/global mutation at module top level.
- **Trait-init family is internally consistent.** `init*Trait` / `init*RuntimeTrait` are uniformly named and uniformly shaped (`(target, obj?/methods?) => void`, defaulting each field), matching the entity/runtime split — runtime traits seed runtime caches, entity traits seed entity fields.
