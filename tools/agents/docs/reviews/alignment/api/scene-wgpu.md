# API Alignment: @flighthq/scene-wgpu

**Verdict:** Strongly conformant — backend-prefixed names, full type words, `Readonly<>` discipline, alias-safe out-params, sentinel returns (no throws), and clean `import type` lines; only a handful of low/medium naming-symmetry nits, the sharpest being a `get*` that hands back mutable shared scratch.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `getWgpuPbrMaterialScratch()` | A `get*` accessor that returns the module-private mutable `Float32Array` (`_materialScratch`) so extension renderers can write extension-factor slots into it. `get*` is reserved for read accessors; this exposes shared mutable state for callers to mutate, and there is no sibling in `scene-gl` (`getGlPbrMaterialScratch` does not exist), so it is a wgpu-only divergence. | Rename to signal mutable-shared-buffer access (e.g. `acquireWgpuPbrMaterialScratch` if framed as a pool-bracket, or fold the extension-factor packing into a dedicated `writeWgpuPbr*Block(out, …)` out-param API so the scratch never leaks as a public getter). Confirm `scene-gl` packs extensions the same way and mirror whichever shape wins. |
| Low | `bindWgpuWireframeColor` | Sibling odd-one-out: the bind family is `bindWgpu{Classic,Debug,Matcap,Toon,Unlit}Surface` (5×) plus this `…Color`. Within the package the `…Surface` suffix is the dominant pattern for "build the material bind group". (No strict cross-package contract: `scene-gl`'s `bindGl*` family is itself varied — `bindGlDebugRange`, `bindGlPbrStandardBlock` — so this is intra-package symmetry only.) | If the wireframe bind group is conceptually the same "material surface" handoff as the others, rename to `bindWgpuWireframeSurface` for family symmetry; keep `…Color` only if it binds strictly a color uniform and nothing surface-shaped. |
| Low | `getWgpuSceneRuntime` | Lazily _allocates_ the runtime on first use (a get-or-create), yet other lazy-allocating caches in the same package use `ensure*` (`ensureWgpuPbrPipeline`, `ensureWgpuMeshUpload`, `ensureWgpuFrameBindGroup`, `ensureWgpuSceneLayouts`). So `get*` vs `ensure*` split on the same "resolve-or-create" concept. Note: this matches the `scene-gl` sibling `getGlSceneRuntime`, so it is a codebase-wide entity/runtime-accessor convention, not wgpu drift — flagging only for the intra-package verb split. | Leave as-is to stay consistent with `getGlSceneRuntime` (entity→runtime resolvers are the documented `get*Runtime` shape); optionally document why runtime resolvers keep `get*` while resource caches use `ensure*` so the split reads as intentional. |
| Low | `getWgpuPbrModuleBody`, `getWgpuWireframeModuleSource`, `getWgpu{Classic,Debug,Matcap,Pbr,Toon,Unlit}ModuleSourceForKey` | Two suffix shapes for "return the WGSL source": the static-body getters end plainly (`…ModuleBody` / `…ModuleSource`) while the keyed variants end `…ModuleSourceForKey`. Minor surface inconsistency in how the same return (a WGSL string) is named. | Acceptable as-is (the `ForKey` suffix usefully marks the specialized vs. static variant). Only worth aligning if a broader module-source naming pass is done across `scene-gl`/`scene-wgpu`. |

## Clean

- **Full, unabbreviated type words throughout.** Every export spells the operand: `compileWgpuPbrPipeline`, `registerStandardPbrWgpuMaterial`, `resolveWgpuMeshMaterialRenderer`, `writeWgpuPbrStandardBlock`. No `Mat`/`Pbr`-only abbreviations of a _type_ (PBR/WGPU are domain acronyms, not shortened type names).
- **Backend prefix-first, consistently.** `Wgpu` leads every name (`drawWgpuScene`, `ensureWgpuMeshUpload`, `bindWgpuToonSurface`), matching the backend-variant-package convention; no `…Wgpu` suffix drift in functions.
- **Allocation verbs match behavior.** `create*` allocates (`createWgpuMeshPipeline`); `ensure*` is get-or-create over a cache; `compile*`/`build*` produce values; `write*`/`upload*` push into existing GPU buffers; no `create*` that fails to allocate.
- **Out-params are alias-safe.** `writeWgpuPbrStandardBlock(out, standard, alphaCutoff)` reads only from `standard`/`alphaCutoff` and writes only to `out` — distinct objects, no read-after-write hazard. The neutral-default branch likewise only writes `out`.
- **`Readonly<>` discipline.** Every object/array input that is not mutated is `Readonly<>` (`Readonly<WgpuPbrDefineKey>`, `Readonly<Material> | null`, `Readonly<Texture> | null`, `Readonly<SceneRenderProxy>`, `Readonly<LinearColor>`); the only bare-mutable object params are the deliberate `out: Float32Array` and the `WgpuRenderState` device handle.
- **Sentinels, never throws for expected-missing.** No `throw` anywhere in non-test source. Lookups return `null` (`getWgpuMeshMaterialRenderer`, `resolveWgpuMeshMaterialRenderer`, `ensureWgpuMeshUpload`, `ensureWgpuWireframeUpload`); booleans use `has*` (`hasWgpuMaterialTexture`).
- **Boolean/accessor prefixes correct.** `hasWgpuMaterialTexture` returns `boolean`; all `get*` return values (no boolean-returning `get*`).
- **Cross-package types stay in the header.** Local exported interfaces (`WgpuMeshPipeline`, `WgpuPbrDefineKey`, `WgpuMeshUpload`, `WgpuMaterialBinding`, `WgpuSceneRuntime`) are renderer-internal and intentionally _not_ in `@flighthq/types`; the header's runtime slots (`sceneMeshUploadCache: WeakMap<object, object>`) stay opaque, so internals never leak upward. Shared contracts (`WgpuMeshMaterialRenderer`, `WgpuRenderState`, `Material`, `Camera`) are imported from `@flighthq/types`.
- **`import type {}` on its own line.** No mixed `import { type Foo, bar }` anywhere in source.
- **Register opt-in, no top-level side effects.** Each material is a `register*WgpuMaterial(state)` opt-in over `registerWgpuMeshMaterialRenderer`; `"sideEffects": false` holds (nothing registers at module load).
- **Consistent parameter ordering.** `(state, key, format)` for every `compile*`/`ensure*Pipeline`; `(state, pipeline, materialKey, …factors)` for every `bindWgpu*`; `(state, …)` leads uniformly.
