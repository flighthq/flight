# API Alignment: @flighthq/displayobject-gl

**Verdict:** Strong overall — full unabbreviated type words, consistent `gl*` prefixing, correct sentinels, and proper `Readonly<>` on inputs; the only real deviations are a `release*`/`destroy*` teardown-verb collision, two cross-package types defined inline instead of in `@flighthq/types`, and one `enable*` that mutates module-global state with no `state` parameter.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| High | `releaseGlRenderCache` (glCache.ts) | Named with the pool-bracket verb `release*`, but it frees a non-GC GPU resource — its own body calls `destroyGlRenderTarget` with the comment "A GlRenderTarget owns a framebuffer and texture; GC will not free them." Per the teardown-verb rule, freeing a GPU resource now is `destroy*`; `release*` is reserved for `acquire*`/`release*` pool brackets, and there is no matching `acquire*`. The verb misdescribes the operation. | Rename to `destroyGlRenderCacheTarget` (it disposes the cache's GPU target, mirroring `destroyGlRenderTarget`). Check the canvas sibling (`releaseCanvasRenderCache`) for the same drift and align both. |
| Medium | `GlShapeMesh` (glShapeMesh.ts) | Cross-package public type defined inline (`export interface GlShapeMesh`). It is the element type of the exported `drawGlShapeMeshes(state, renderProxy, meshes: readonly GlShapeMesh[])` parameter, so it crosses the package boundary as public API. The map requires such shapes live in `@flighthq/types` (the "header layer"); the sibling `GlVelocityWriter`/`GlVelocityContext` already follow this. | Move `GlShapeMesh` to `@flighthq/types` and import it. |
| Medium | `GlRichTextOverlay` (glRichText.ts) | Cross-package public type defined inline (`export type GlRichTextOverlay = (...) => void`). It is a parameter of the exported `registerGlTextInputOverlay` and `drawGlRichTextWithOverlay`, so it is part of the public seam consumed by `glTextInput.ts`. Same rule as above. | Move `GlRichTextOverlay` to `@flighthq/types`. |
| Medium | `enableGlTextInput()` (glTextInput.ts) | (1) Parameter asymmetry: every other `enable*`/`register*` in this package takes `state: GlRenderState` (`enableGlClipSupport`, `enableGlRenderCache`, `registerGlVelocityWriter`, …) but this one takes no args. (2) It registers into module-global mutable state — `registerGlTextInputOverlay` stores `_webglTextInputOverlay` at module scope in glRichText.ts. The map forbids shared top-level mutable state for opt-in subsystems; the overlay should hang off the render state's runtime like the other registrations, so two states cannot clobber each other's overlay. | Make it `enableGlTextInput(state)` and store the overlay on the GL render-state runtime instead of a module global, matching the rest of the registration surface. |
| Low | `registerGlTextInputOverlay` / `_webglTextInputOverlay` | Naming token drift: the internal global is `_webglTextInputOverlay` (`webgl` prefix) while the package's entire public surface uses the `gl*` token. Inconsistent backend token within the package. | Rename the internal to `_glTextInputOverlay` for token consistency (fold into the runtime-slot fix above). |
| Low | `remapGlScale9Commands(out, source, mapper)` (glScale9Shape.ts) | Public out-param function typed `out: unknown[]`, `source: readonly unknown[]` — the command stream has no real type, so the signature carries no information and `out` is read-after-write in the post-process loop (safe only because `out` and `source` are distinct here; the alias-safety contract is unstated). Mirrors the weak `unknown[]` typing of the canvas command path. | Type the command array (a `Scale9ShapeCommand[]` / shared command type in `@flighthq/types`) rather than `unknown[]`; document that `out` must not alias `source`. |
| Low | `drawGlTextInputOverlay(context, source, result, fieldW, fieldH, _text)` | Trailing `_text: string` is an unused parameter on a public function (underscore-prefixed = intentionally unused), and `fieldW`/`fieldH` abbreviate width/height where the rest of the package spells them out (`width`, `height`). Dead public parameter plus local abbreviation. | Drop `_text` from the public signature; rename `fieldW`/`fieldH` to `fieldWidth`/`fieldHeight`. |

## Clean

- **Full, unabbreviated type words throughout.** `getDisplayObjectBounds`-style discipline holds: `renderGlDisplayObject`, `drawGlParticleEmitter`, `getGlRenderProxyColorTransform`, `registerGlUniformColorTransformMaterial`, `buildGlScale9Mapper` — no `DO`/`Obj`/`Xform` abbreviations of type words.
- **Consistent `gl*` backend prefix** on every exported function (prefix-first), and the canvas re-exports are aliased into the `gl*` namespace (`registerCanvasShapeCommands as registerGlShapeCommands`), keeping root names globally unique.
- **Correct sentinels for expected failure.** `buildGlScale9Mapper` returns `null` on non-positive scale/bounds; `getGlRenderCacheTarget`, `getGlVelocityWriter`, `getGlRenderProxyColorTransform`, `createGlScale9ShapeData` all return `… | null`. The only `throw`s (`glMaterials.ts`, `glVelocity.ts`) are GL shader compile/link failures, an acceptable build-time invariant, not validation of expected-missing cases.
- **Allocation discipline by verb.** `create*` allocate (`createGlCacheState`, `createGlRenderTarget` usage, `createGlVelocityTarget`, `createGlRichTextData`); `ensure*` allocate-or-reuse (`ensureGlRenderCacheTarget`, `ensureGlQuadBatchShader`); hot-path draw/`prepare`/`pack`/`set` helpers write into existing buffers/state without returning fresh allocations.
- **`destroy*` used correctly elsewhere.** `destroyGlRichTextData`, `destroyGlScale9ShapeData`, `destroyGlVideoData` all free a GPU texture/target now — the right verb (making the `releaseGlRenderCache` outlier stand out).
- **`Readonly<>` on non-mutated object inputs:** `buildGlScale9Mapper` (both rectangles), `getGlRenderProxyColorTransform` (`Readonly<RenderProxy>`), `drawGlVelocityQuad` (`Readonly<GlVelocityContext>`), `pushGlClipRectangle`/`pushGlClipContours` (transforms), `renderGlVelocity` (`Readonly<Transform2DNode>`, `Readonly<GlRenderTarget>`).
- **`get*` reserved for accessors; no boolean `get*`.** `refreshGlRenderCache` returns `boolean` but is correctly a verb, not a `get*`. No `get*` returns a boolean.
- **`import type {}` on its own line** in every file; no mixed `import { type Foo, bar }`. The shared `GlVelocityWriter`/`GlVelocityContext` types are sourced from `@flighthq/types`, the correct header layer (precedent the two inline-type findings should follow).
