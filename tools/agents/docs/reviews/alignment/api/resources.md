# API Alignment: @flighthq/resources

**Verdict:** Mostly aligned — full type words and `Readonly`/`out`/teardown discipline are solid, but a package-wide `Url`↔`URLs` acronym-casing split, two `throw`s on expected-missing cases, and `Partial<Entity>` vs `Partial<*Like>` constructor drift need cleanup.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| High | `createAudioResourceFromUrl` / `createAudioResourceFromURLs` (also `loadAudioResourceFromUrl`/`FromURLs`, `createVideoResourceFromUrl`/`FromURLs`, `loadVideoResourceFromUrl`/`FromURLs`, `loadFontFromUrl`/`FromURLs`, `loadFontResourceFromUrl`/`FromURLs`) | The same acronym is cased two ways within one package: single-source uses `Url` (camelCase), multi-source uses `URLs` (all-caps). Globally-unique names are fine, but the inconsistent capitalization of one acronym breaks the "same concept, same spelling" rule and is a constant grep/recall hazard. The image/atlas/tileset side has no multi-URL variant, so the only place the plural appears it disagrees with the singular. | Pick one casing for the acronym package-wide and apply it to both arities. Recommend `Url`/`Urls` (matches the existing singular `FromUrl` and Flight's "no abbreviation gymnastics" tone): `createAudioResourceFromUrls`, `loadFontFromUrls`, etc. |
| High | `loadImageResourceFromArrayBuffer` | `throw new Error('Unable to determine image type…')` for an expected-failure case (caller passed bytes whose format can't be sniffed and gave no `mimeType`). Convention reserves `throw` for programmer error; undetectable input is an expected runtime outcome. | Return a sentinel: change the signature to `Promise<ImageResource \| null>` and return `null` when `type === null`, mirroring `detectImageMimeType`'s own `string \| null` contract. |
| Medium | `getAudioContext` | `throw new Error('AudioContext is not available.')` for an environment-unavailable case. The platform-suite rule is that web/unsupported paths return sentinels (`null`) rather than throwing; absence of `AudioContext` is environment unavailability, not API misuse. | Return `AudioContext \| null` and return `null` when no constructor exists, letting callers branch instead of catching. |
| Medium | `buildTilesetRegions` | `build*` is outside the package's verb vocabulary (`create*`/`clone*`/`set*`/`add*`/`invalidate*`/`dispose*`). It mutates `target` in place (rewrites `atlas.regions`), so it is an update, not a builder/allocator. | Rename to `updateTilesetRegions` (or `rebuildTilesetRegions`) to signal in-place mutation of an existing tileset and align with the update-verb family. |
| Medium | `createTextureAtlas` (param `Partial<TextureAtlas>`), `createTileset` (`Partial<Tileset>`) | Constructors accept `Partial<Entity>` while the sibling `createTextureAtlasRegion` correctly accepts `Partial<TextureAtlasRegionLike>`. The map says structural literals are only for `*Like` inputs; an entity type carries runtime/binding identity its `Partial` does not. Inconsistent within the same file family. | Introduce/accept `Partial<TextureAtlasLike>` and `Partial<TilesetLike>` (define the `*Like` shapes in `@flighthq/types`), matching the region constructor. |
| Low | `detectImageMimeType` | Accessor-style function returning a value uses `detect*`, outside the `get*`/`has*`/`is*` accessor vocabulary. It is a pure derive-from-input getter. | Acceptable as a domain verb, but consider `getImageMimeType` for accessor symmetry; if kept, treat `detect*` as a deliberate, documented sniff-verb. |
| Low | `setTextureAtlasRegion(out, x, y = 0, width = 0, …)` | Asymmetric defaults: `x` is required while `y`/`width`/`height`/`pivotX`/`pivotY` default to `0`. A setter that defaults every field but the first is surprising; either all geometry args are required or all default. | Drop the default on the trailing args (require the rect) or also default `x` to `0`, for a uniform signature. |
| Low | `getAudioContext` (module-level `let context`) | Shared module-level mutable singleton. Import stays side-effect-free (lazy init inside the function), so it does not violate `sideEffects: false`, but it is package-global mutable state, which the map discourages. | Acceptable as a lazy singleton; note it as the one shared-state slot. No change required if intentional. |

## Clean

- **Full, unabbreviated type words everywhere.** Every exported name spells out its type: `addTextureAtlasRegionRectangleXY`, `createImageResourceFromImageBitmap`, `loadTilesetFromArrayBuffer`. No `Tex`/`Img`/`Res` abbreviations. This is the package's strongest dimension.
- **Globally self-identifying, unique names.** Each function leads straight back to its domain (`ImageResource`, `TextureAtlas`, `Tileset`, `Font`, `AudioResource`, `VideoResource`); no bare `load`/`create` that would collide across the barrel.
- **Allocation-by-verb discipline.** `create*`/`clone*` allocate; in-place writers use `out`/`target` (`setTextureAtlasRegion`, `loadFontResourceFrom*`, `addTextureAtlasRegion*`) and return `void` or the mutated `out`.
- **Alias-safety.** `setTextureAtlasRegion` writes only `out` fields from scalar args (no input aliasing); `cloneImageResource` reads all source fields into the new entity before any write — no read-after-write hazards.
- **Teardown verbs correct.** `disposeImageResource` releases `data`/`source` references to GC and explicitly documents that it does **not** free the GPU texture (a `destroy*` concern) — the `dispose*`/`destroy*` distinction is honored and commented.
- **`Readonly<T>` on read-only params.** Query/clone functions (`hasImageResourceData`, `hasImageResourceSource`, `isImageResourceEmpty`, `cloneImageResource`, `addTextureAtlasRegionRectangle`/`Vector2`) take `Readonly<…>`; mutators (`setImageResourceSource`, `invalidateImageResource`, `addTextureAtlasRegion`) correctly take mutable references.
- **Accessor/boolean prefixes correct.** `get*` for value accessors (`getAudioContext`), `has*`/`is*` for booleans (`hasImageResourceData`, `hasImageResourceSource`, `isImageResourceEmpty`, `isImageResourceSameOrigin`).
- **Sentinels in the right places.** `detectImageMimeType` returns `string \| null`, `createAudioResourceFromURLs`/`loadAudioResourceFromURLs` return a default resource (not a throw) when no source is playable.
- **`import type {}` hygiene.** Every file isolates type imports on dedicated `import type { … } from '@flighthq/types'` lines; no inline `import { type Foo, bar }` mixing. Cross-package types (`ImageResource`, `TextureAtlas`, `FontUrl`, `RectangleLike`, `Vector2Like`, etc.) all come from `@flighthq/types`, none defined inline.
