# API Alignment: @flighthq/entity

**Verdict:** Naming, allocation verbs, and unabbreviated type words are all clean; the one real defect is that `getEntityRuntime` is typed non-null but returns `undefined` for a runtime-less entity (a sentinel-rule violation that also propagates into `@flighthq/node`), with a few smaller symmetry nits around `attach` having no `detach` and inconsistent `Readonly`/parameter naming.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- | --- |
| High | `getEntityRuntime` | Signature is `getEntityRuntime(source: Readonly<Entity>): Readonly<EntityRuntime>` but the body is `source[EntityRuntimeKey]!` — a fresh `createEntity()` has `undefined` runtime, so the function returns `undefined` while the type promises a value. The package's own tests assert this (`runtime.test.ts:22` and `binding.test.ts:10` both `expect(getEntityRuntime(entity)).toBeUndefined()`). The non-null `!` launders an expected-missing case into a lie. This violates the sentinel rule (return `null`/`undefined` for expected-missing, type it honestly) and the rule against unreachable-invariant assertions used to silence the compiler. | Type the missing case: return `Readonly<EntityRuntime> | null`and`return source[EntityRuntimeKey] ?? null`(matching`getEntityBinding`, which correctly returns `object \| null`). If a non-null variant is genuinely wanted for callers that have already attached a runtime, add a separate `ensureEntityRuntime`/lazy accessor rather than asserting. |
| Medium | `getEntityRuntime` (downstream) | The same lie is inherited verbatim by `@flighthq/node`: `getNodeRuntime` (`packages/node/src/node.ts:87`) and every `getEntityRuntime(...) as ...!` callsite in `node.ts`/`boundsRectangle.ts`/`transform3d.ts` build on the false non-null contract. Fixing only the local type will surface real ` | null`handling in`node`. | Fix the base contract first, then thread the nullability (or an explicit `ensure*`) through `@flighthq/node` rather than re-asserting at each callsite. Flag to the user as a cross-package follow-up since it reaches outside this worktree's package. |
| Low | `attachEntityBinding` | `attach*` implies a paired `detach*` (the platform suite's event-wiring convention is `attach*`/`detach*`), but there is no `detachEntityBinding`/`clearEntityBinding`. The binding slot can be set but never cleared through the public API. | Add `detachEntityBinding(entity)` (or `clearEntityBinding`) that nulls the slot, or rename to a verb that does not imply a teardown counterpart (e.g. `setEntityBinding`) for symmetry with `getEntityBinding`. The `set*`/`get*` pairing also reads cleaner than `attach*`/`get*`. |
| Low | `attachEntityBinding`, `getEntityBinding` | Parameter naming is inconsistent with the accessor pair: `attachEntityBinding(entity, ...)` uses `entity`, while `getEntityBinding(source)` and `getEntityRuntime(source)` use `source`. | Pick one. Use `entity` everywhere for this package's primitive (clearer than the generic `source`), or `source` everywhere; do not mix within four functions. |
| Low | `getEntityBinding` | Takes non-`Readonly` `source: Entity` despite being a pure read accessor; `getEntityRuntime` correctly takes `Readonly<Entity>`. Asymmetric. | Change to `getEntityBinding(source: Readonly<Entity>)` to match `getEntityRuntime` and the Readonly-by-default rule for read-only parameters. |

## Clean

- **Full, unabbreviated type words.** Every exported name spells out `Entity` / `EntityRuntime` / `EntityBinding` in full (`createEntity`, `createEntityRuntime`, `getEntityRuntime`, `attachEntityBinding`, `getEntityBinding`). No abbreviation, no `Obj`/`Rt`-style shortening.
- **Globally unique root barrel.** None of the five exports collide with sibling-package roots; `npm run api entity` shows no name clashes (the unrelated `*Identity*` matches are substring noise, not collisions).
- **Allocation discipline by verb.** The two allocators are correctly named `create*` (`createEntity`, `createEntityRuntime`); the accessors (`getEntityRuntime`, `getEntityBinding`) and the mutator (`attachEntityBinding`) allocate nothing in the read path. No hot-path helper allocates.
- **Accessor prefixes.** `get*` is used for the two accessors and both return values, not booleans; there are no boolean-returning `get*` and no missing `is*`/`has*`.
- **Sentinel done right (one of two).** `getEntityBinding` returns `object | null` and genuinely returns `null` for both no-runtime and empty-slot cases — the correct model that `getEntityRuntime` should mirror.
- **No thrown errors.** No `throw` for expected-missing cases anywhere in the package.
- **Types sourced from the header layer.** `Entity`, `EntityRuntime`, and `EntityRuntimeKey` are all imported from `@flighthq/types`; nothing cross-package is defined inline. `import type {}` is on its own line in every source file (`entity.ts`, `binding.ts`, `runtime.ts`), separate from the value import of `EntityRuntimeKey`.
- **Entity/runtime split honored.** The package is the canonical expression of the map's entity↔runtime pattern: the entity holds only the opaque `[EntityRuntimeKey]` slot, runtime state (`binding`) lives on the runtime object, and `createEntity` leaves the runtime `undefined` (lazy) rather than eagerly allocating.
- **Alphabetized exports.** Within each file exports are ordered (`createEntityRuntime` before `getEntityRuntime`; `attachEntityBinding` before `getEntityBinding`); `npm run order` has nothing to flag.
- **No top-level side effects.** `"sideEffects": false`, single `.` export, thin barrel re-exporting the three modules — fully tree-shakable, no module-top registration or mutable state.
