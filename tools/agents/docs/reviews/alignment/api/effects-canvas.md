# API Alignment: @flighthq/effects-canvas

**Verdict:** Strongly conformant — naming, allocation verbs, pool brackets, and Readonly inputs are disciplined and consistent; the one real defect is `destroyCanvasRenderEffectPipeline` using `destroy*` for a GC-only teardown, plus a recurring `Readonly<>` dressing on the mutated `dest` output param.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `destroyCanvasRenderEffectPipeline` | Verb misuse. The map defines `destroy*` as "free a non-GC resource now (GPU/native handle)" and `dispose*` as "detach/release-to-GC." This function's own comment states "Canvas elements are plain GC-managed memory with no GPU handles to free; drop references so the pool and scene canvas become eligible for collection" — that is the definition of `dispose*`. The `destroy*` verb is inherited from the Gl/Wgpu siblings (`destroyGlRenderEffectPipeline`, `destroyWgpuRenderEffectPipeline`), which genuinely free GPU framebuffers/textures. Cross-backend name symmetry is in tension with the verb-by-semantics rule; here semantics should win since nothing non-GC is freed. | Rename to `disposeCanvasRenderEffectPipeline`. The asymmetry with the GPU siblings is correct: they `destroy`, this disposes, because that is exactly what each does. |
| Low | `apply*EffectToCanvas` (all ~45) and `drawCanvasEffectPass` / `passthroughCanvasEffectPass` | The output target `dest` is typed `Readonly<CanvasRenderTarget>` yet is the write target (the functions mutate `dest.context` via `drawImage`/`clearRect`). The map says mutable outputs "are usually named `out` or `target`" and reserves `Readonly<>` for params not intended to be mutated. The reference is not reassigned, but its pixels are the entire point of the call, so labeling it `Readonly` and naming it `dest` (not `out`/`target`) understates that it is the mutated output. | Acceptable as-is if treated as a deliberate "the handle is const, its backing store is the output" convention shared across effects-gl/wgpu — but then document that convention once. Otherwise drop `Readonly<>` on `dest` and/or rename `dest` → `target` to mark it as the output, consistently across the three effects backends. |
| Low | `createCanvasRenderEffectPipeline(_state, ...)` | Leading `_state` param is unused (accepted only for signature parity with the Gl pipeline). A `create*` that ignores its first argument is slightly surprising at the call site. | Keep for cross-backend parity, but this is worth a one-line note in the doc comment (the comment currently explains the ignored `options.*` fields but not the ignored `_state`). |
| Info | `applyCrtEffectToCanvas`, `applyFxaaEffectToCanvas`, `applySmaaEffectToCanvas`, `applySsaoEffectToCanvas`, `applySsrEffectToCanvas`, `applyTaaEffectToCanvas` | These embed acronyms (`Crt`, `Fxaa`, `Smaa`, `Ssao`, `Ssr`, `Taa`) rather than spelled-out words. Not a local violation — each is the exact, full type word from `@flighthq/types` (`CrtEffect`, `FxaaEffect`, …); the function name must track the type name. Flag belongs upstream in `@flighthq/effects`/`@flighthq/types`, not here. | No change in this package. If the acronyms are ever reconsidered, do it at the type definition and let all backends follow. |

## Clean

- **Full type words.** Every `apply<X>EffectToCanvas` and `defaultCanvas<X>EffectRunner` carries the complete, unabbreviated effect type word matching its `@flighthq/types` source; nothing is locally truncated (`getDOBounds`-style abuses absent).
- **Globally unique, backend-tagged names.** Every export is prefixed/suffixed with `Canvas` (`acquireCanvasRenderTarget`, `getCanvasRenderEffectRunner`, `defaultCanvas…Runner`), so root-barrel names do not collide with the Gl/Wgpu siblings.
- **Allocation by verb.** `create*` allocates (`createCanvasRenderEffectPipeline`, `createCanvasRenderTargetPool`); `acquireCanvasRenderTarget` may allocate or reuse from the pool; the hot-path `apply*`/`draw*` functions write into the `dest` target and return `void` — no per-call return allocation.
- **Pool brackets.** `acquireCanvasRenderTarget` / `releaseCanvasRenderTarget` are a correct matched pair; `release*` is used only for pool return (not as a teardown synonym), and multi-pass recipes (`applyBloomEffectToCanvas`, film grain, pixelate) acquire and release symmetrically.
- **Sentinels, no defensive throws.** `getCanvasRenderEffectRunner` returns `null` for a missing kind (expected-miss → sentinel); the package throws nowhere. `endCanvasRenderEffectPipeline` skips an unregistered effect kind (`runner === null → continue`) rather than throwing.
- **Accessor naming.** `getCanvasRenderEffectRunner` is a true `get*` returning a value-or-`null`, not a boolean; no `get*`-returns-boolean defects.
- **Readonly inputs.** Read-only inputs are consistently `Readonly<>`: `source`, `effect`, `options`, and `effects: ReadonlyArray<RenderEffect>` in the pipeline. (The only `Readonly<>` tension is the _output_ `dest`, noted above.)
- **Import & type hygiene.** All cross-package types come from `@flighthq/types` (none redefined inline); every `import type { … }` is on its own dedicated line — no `import { type Foo, bar }` mixing anywhere in the package.
- **Alias-safety / mutation boundary.** Effect functions read `effect.*` fields into locals before drawing, and `drawCanvasEffectPass` fully resets canvas state (transform/alpha/composite/filter, then `clearRect`) before writing, so a recipe never inherits or clobbers caller state mid-pass.
- **Tree-shakable, side-effect-free dispatch.** The registry is a per-state `WeakMap` populated only by explicit `registerCanvasRenderEffect`; no top-level registration, matching `"sideEffects": false`. Each effect's `default*Runner` is a separate export a caller opts into.
