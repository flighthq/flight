# API Alignment: @flighthq/shape

**Verdict:** Strong overall — full type words, consistent `append*`/`create*`/`compute*` families, and clean `import type` discipline; two real issues: `getShapeFillRegions` is a `get*` that allocates (should be `create*`), and `copyShapeCommands` is not alias-safe.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| High | `getShapeFillRegions` | A `get*` accessor that **allocates**: builds and returns a fresh `ShapeFillRegion[]` plus new `Path`/region objects each call. Violates the allocation-by-verb rule (`get*` is a non-allocating accessor; allocation is `create*`) and the verb-consistency rule — every sibling that produces a region uses `create*`: `createClipRegionFromPath`, `createClipRegionFromRectangle`, `createSurfaceRegion`, `createTextureAtlasRegion`. | Rename to `createShapeFillRegions`. It returns `null` for the not-expressible case, which is a fine sentinel; the rename only fixes the verb. |
| Medium | `copyShapeCommands` | Not alias-safe. Body does `target.data.commands.length = 0` then `target.data.commands.push(...source.data.commands)`; when `source === target` it zeroes the array first, then spreads the now-empty array, silently clearing the shape. The CLAUDE map requires out-param/copy functions to be alias-safe unless documented otherwise (read inputs into locals before writing). | Snapshot the source first: `const cmds = source.data.commands.slice(); target.data.commands.length = 0; target.data.commands.push(...cmds);` — or early-return when `source === target`. |
| Low | `computeShapeLocalBoundsRectangle` | Hot-path bounds function allocates two closures (`expand`, `quadPoint`) on every call. The function is correctly `out`-first and matches the sibling signature, but per the "bounds functions must not allocate" intent these inner closures are per-call allocations in a measure path. | Hoist `expand`/`quadPoint` to file-scope free functions (passing the accumulator min/max via locals or a small scratch struct), or inline them. Purely an allocation-discipline nit. |
| Low | `getShapeFillRegions`, `hasNonSolidShapeFill` | Public parameter typed `readonly unknown[]` (the raw command stream) rather than a named shared type. The command stream is the shape's core data contract and is duplicated as `unknown[]` in `hitTestShapeCommandPoint(buf: unknown[])`. Cross-package consumers (`displayobject-canvas` `renderCanvasShapeCommands(commands: unknown[])`) re-type it ad hoc. | Consider a named `ShapeCommandStream` (or `readonly ShapeCommand[]`) alias in `@flighthq/types` so the stream has one navigable type instead of bare `unknown[]` repeated across packages. Type-precision improvement, not a blocker. |

## Clean

- **Full, unabbreviated type words throughout.** Every export spells out its operand: `appendShapeBeginGradientFill`, `computeShapeLocalBoundsRectangle`, `createScale9ShapeRuntime`, `hitTestShapeCommandPoint`, `registerShapeHitTestCommand`. No abbreviations.
- **Globally unique root names.** No collision with the renderer packages, which correctly carry backend tokens (`drawCanvasShape`, `drawGlShape`, `drawWgpuShape`) — the core package owns the unprefixed names.
- **`create*` allocation verb used correctly** for the entity/data/runtime quartet: `createShape`, `createShapeData`, `createShapeRuntime`, and the `createScale9Shape*` set. `copyShapeCommands` uses `copy` consistent with `copyMatrix`/`copyRectangle`, with `(source, target)` order matching siblings.
- **`compute*` out-param signature matches every sibling exactly:** `computeShapeLocalBoundsRectangle(out: Rectangle, source: Readonly<Node>)` mirrors `computeBitmapLocalBoundsRectangle`, `computeSpriteLocalBoundsRectangle`, etc. — same name shape, same `out`-first parameter order, same `Readonly<Node>` source.
- **`get*` accessors are genuine non-allocating getters** (`getShapeRuntime`, `getScale9ShapeRuntime`) returning `Readonly<...Runtime>`. (The one exception, `getShapeFillRegions`, is flagged above.)
- **Sentinels over throws.** `getShapeFillRegions` returns `null` when the shape is not solid-fill expressible; `hitTestShapeCommandPoint` returns `null` for an unregistered command key. No thrown errors for expected-missing cases.
- **Boolean predicate correctly prefixed:** `hasNonSolidShapeFill` returns `boolean` and uses `has*`.
- **`Readonly<T>` applied on inputs that are not mutated** (`copyShapeCommands` source, `getShapeRuntime`/`getScale9ShapeRuntime` source, `createShape`/`createShapeData` obj/data, `createScale9Shape` `scale9Grid`). The `append*`/`clear*`/`invalidate*` mutators correctly take a mutable `Shape`.
- **`*Like` used for structural input:** `createScale9Shape(scale9Grid: Readonly<RectangleLike>)` accepts a structural grid input rather than forcing an entity `Rectangle`.
- **Cross-package types sourced from `@flighthq/types`** (`Shape`, `ShapeData`, `ShapeRuntime`, `Scale9Shape*`, `ShapeFillRegion`, `ShapeHitTestCommand`, `ShapeCommandKey`, gradient/caps/joint enums); none defined inline. `import type {}` is on its own line in every file. `PathCommand` is re-exported from types with a clear comment for a single authoring import surface.
- **Registry seam follows the kind/registration pattern:** `registerShapeHitTestCommand` is a `register*` opt-in (no top-level side effects), generic over `ShapeCommandKey`, matching `registerCanvasShapeCommand` in the renderer packages.
