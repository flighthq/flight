# API Alignment: @flighthq/interaction

**Verdict:** Strong adherence overall — names carry full type words, sentinels are used for misses, and the `enable*`/`get*` signal pair matches siblings; the only real defects are stale doc references to a renamed export and a missing `Readonly<>` on a pure-read accessor.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `registerHitTestPoint` | The function's own JSDoc says "register an interaction handler … via `registerHitTest`" and `hitTestGraphPoint`'s JSDoc also says "must be registered via `registerHitTest`" — but the exported name is `registerHitTestPoint`. Stale reference to a prior name; a reader grepping for `registerHitTest` finds nothing. | Update both JSDoc blocks to `registerHitTestPoint`. |
| Low | `getInteractionSignals` | Pure read accessor takes `source: N` (mutable) while the sibling pattern `getStageSignals(source: Readonly<Stage>)` and the package's own internal `getInteractionSignal(source: Readonly<N>)` mark the source `Readonly`. Violates the "Readonly wherever mutation isn't intended" rule for a getter. | Change signature to `getInteractionSignals<N extends NodeAny>(source: Readonly<N>)`. (`enableInteractionSignals` legitimately stays mutable — it lazily writes `runtime.interactionSignals`.) |
| Low | `hitTestDisplayObjects` | JSDoc references a parameter `obj` ("overlaps … with the bounding box of the `obj` display object") but the parameter is named `other`. Stale doc drift from a renamed param. | Update JSDoc to `other`. |
| Low | `captureInteractionPointer` / `releaseInteractionPointer` | `release*` is documented as reserved for pool/cache `acquire`/`capture` brackets (per the teardown-verb rule). Here `capture`/`release` mean pointer capture (DOM `setPointerCapture`/`releasePointerCapture` semantics), not pool return. The domain term is correct and industry-standard, but it collides with the reserved-verb rule. | Keep the names (they match the universally understood pointer-capture vocabulary); optionally note in the verb-reservation rule that `capture`/`release` for _pointer capture_ is a sanctioned domain exception so future audits don't re-flag it. |
| Low | `connectInputToInteraction` (`coordScale`) | `coordScale` abbreviates "coordinate", and parameter names are part of the public surface. Minor, but the codebase rule is to avoid abbreviating. | Rename to `coordinateScale`. |
| Low | `connectInputToInteraction` vs `connectInteractionSignal` | Teardown asymmetry: `connectInputToInteraction` returns a disposer (`() => void`), while `connectInteractionSignal` requires a separate `disconnectInteractionSignal(...)` call with the same args. Two different disconnection idioms for two `connect*` functions in the same package. | Acceptable if intentional (signal connect/disconnect mirrors the `@flighthq/signals` paired API), but consider documenting why one returns a disposer and the other does not. |
| Low | `defaultDisplayObjectHitTestPoint`, `defaultStageHitTestPoint`, `defaultMovieClipHitTestPoint`, `defaultHtmlViewHitTestPoint` | Naming and signatures are clean, but `defaultQuadBatchHitTestPoint` and `defaultTilemapHitTestPoint` are `// TODO` stubs that just delegate to `defaultSpriteHitTestPoint`. Exported but incomplete — a completeness gap, not a naming defect. | Track the QuadBatch/Tilemap precise hit-test as follow-up work; the export shape itself is fine. |

## Clean

- **Full, unabbreviated type words everywhere.** `findGraphHitTarget`, `hitTestGraphLocalBounds`, `hitTestGraphPoint`, `hitTestDisplayObjects`, `dispatchInteractionPointerDown`, `enableInteractionSignals`, and every `default*HitTestPoint` spell out their full subject — no `getDOBounds`-style abbreviation.
- **`enable*Signals` / `get*Signals` pair** matches the sibling convention exactly (`enableStageSignals`/`getStageSignals`, `enableNodeSignals`/`getNodeSignals`). `enable*` is the opt-in signal-group gate the map prescribes.
- **Sentinels for expected misses.** `findGraphHitTarget` and `getInteractionSignals` return `null` for "nothing hit" / "not enabled"; no throwing for expected-empty cases. No misuse-validation of unreachable internal invariants.
- **`get*` / `has*` / `is*` discipline.** Accessors use `get*`; boolean internals use `has*`/`is*` (`hasInteractionSignalSubscriber`, `isPointerSignalNeeded`, `isTransform2DNode`). No `get*` returning a boolean.
- **`Readonly<>` on options and event-data params** — `Readonly<InteractionManagerOptions>`, `Readonly<InteractionPointerOptions>`, `Readonly<Partial<KeyboardEventData>>`, `Readonly<SignalConnectOptions>` are all marked immutable.
- **Cross-package types sourced from `@flighthq/types`** (`HitTestFunction`, `NodeAny`, `InteractionManager`, `PointerEventData`, etc.); `import type {}` is on its own line in every file, never inline-mixed.
- **`create*` allocates, dispatch/hit-test do not allocate a return value** — `createInteractionManager`/`createInteractionSignals` are the only allocators; hit tests and dispatches reuse module-level scratch (`_pointerData`, `_keyboardData`, `_localPoint`) rather than allocating per call, consistent with the hot-path no-alloc intent (note: that scratch is shared mutable state, fine for synchronous single-dispatch but re-entrancy-fragile — an internal concern, not an exported-API one).
- **Globally unique export names** — every export is prefixed with `Interaction`, `Graph`, `HitTest`, or `default*HitTestPoint`, so no collisions against the SDK barrel.
