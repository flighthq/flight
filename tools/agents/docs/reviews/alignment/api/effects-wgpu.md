# API Alignment: @flighthq/effects-wgpu

**Verdict:** Strong conformance overall — consistent `apply<Effect>EffectToWgpu` family, correct `Readonly<>`/sentinel/teardown verbs, type imports clean — with one real naming collision (`getWgpuEffectPipeline` vs the `WgpuRenderEffectPipeline` entity) and a dead public parameter (`applyTaaEffectToWgpu`'s `_effect`) worth fixing pre-release.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `getWgpuEffectPipeline` | "Pipeline" is overloaded in the barrel. This returns a compiled shader program (`WgpuFilterPipeline`) and lives in `wgpuEffectProgramCache.ts`, but `createWgpuRenderEffectPipeline`/`WgpuRenderEffectPipeline` in the same package use "Pipeline" for the post-process pass entity. Two unrelated concepts share the word at the package root. | Rename to reflect what it returns and its cache role, e.g. `getWgpuEffectFilterPipeline` or `getWgpuEffectProgram` (matching the file `…ProgramCache` and the cited `effects-gl getGlEffectProgram`). The function comment already says it "Mirrors effects-gl's getGlEffectProgram" — align the name to that mirror. |
| Medium | `getWgpuEffectPipeline` | Return-type word mismatch: name says "EffectPipeline" but the declared return is `WgpuFilterPipeline` (from `@flighthq/filters-wgpu`). The full type word of the returned value is not in the function name. | Fold into the rename above so the name carries the returned type's word (`…FilterPipeline` / `…Program`). |
| Low | `applyTaaEffectToWgpu` | Public signature carries an unused, underscore-prefixed parameter `_effect: Readonly<TaaEffect>`. TAA is currently a passthrough; the dead param leaks an implementation gap into the public API. | Keep the parameter name as `effect` (drop the underscore) for family symmetry and reference it once it is implemented; the underscore signals "intentionally unused" which should not appear in an exported signature. Note the stub status in a comment instead. |
| Low | `createWgpuRenderEffectPipeline` | First parameter `_state: WgpuRenderState` is unused (underscore-prefixed) — accepted for symmetry with the Gl mirror, but an exported `create*` taking an ignored required argument is a small API smell. | Acceptable if the state is reserved for future allocation; otherwise drop it. If kept, name it `state` (no underscore) since it is part of the public contract. |
| Info | `applyBloomEffectToWgpu`, `applyMotionBlurEffectToWgpu`, `applyBokehDepthOfFieldEffectToWgpu` | These three break the otherwise-uniform `(state, source, dest, effect)` shape with an extra middle argument (`pool`, `velocityTexture`, and a depth omission respectively). Insertion is consistent (`effect` stays last), and each extra is justified (multi-pass pooling / velocity G-buffer), so this is a documented asymmetry, not a defect. | No change required; the per-runner `WgpuRenderEffectContext` already normalizes these at the registry seam (`ctx.pool`, `ctx.sceneVelocityTexture`). Worth a one-line note that the direct `apply*` form intentionally exposes the extra input. |

## Clean

- **Type-word naming.** Every `apply<Effect>EffectToWgpu` and `defaultWgpu<Effect>EffectRunner` carries the full, unabbreviated upstream type word. The acronym families (`Crt`, `Fxaa`, `Smaa`, `Ssao`, `Ssr`, `Taa`, `BokehDepthOfField`) match their `@flighthq/types` type names exactly — no abbreviation violation.
- **Globally unique names.** The `…ToWgpu` suffix and `defaultWgpu…` prefix make all 46 effect entry points and runners self-identifying and collision-free against the gl/canvas siblings.
- **Allocation/teardown verbs.** `createWgpuRenderEffectPipeline` allocates; `destroyWgpuRenderEffectPipeline` correctly uses `destroy*` (it frees GPU render targets + pool, a non-GC resource). Inside the recipes, pool use is correctly bracketed `acquireWgpuRenderTarget`/`releaseWgpuRenderTarget` (Bloom, pipeline).
- **Sentinels over throws.** `getWgpuRenderEffectRunner` returns `… | null`; unknown effect kinds are skipped in `endWgpuRenderEffectPipeline`; the velocity-absent path in motion blur is a sentinel passthrough. No thrown errors for expected-missing cases.
- **`get*` accessors.** `getWgpuRenderEffectRunner` / `getWgpuEffectPipeline` are genuine getters returning values (not booleans).
- **`Readonly<>` discipline.** All `source`/`dest`/`effect` params and the `RenderEffectPipelineOptions` are `Readonly<>`; `effects` is `ReadonlyArray<RenderEffect>`. Mutable params are limited to the deliberately-mutated `state`, `pipeline`, and `pool`.
- **Registry verbs.** `registerWgpuRenderEffect` matches the kind-based `register*` convention; the registry is per-state `WeakMap`, opt-in, last-write-wins — consistent with the SDK renderer-registration pattern and the effects-gl mirror.
- **Type imports.** All `import type {}` are on their own lines; no inline `import { type Foo, bar }`. Cross-package types come from `@flighthq/types`; the one non-types type in a signature (`WgpuFilterPipeline`) is a legitimately exported sibling type from `@flighthq/filters-wgpu`, a declared dependency.
- **File alignment.** One source file per exported recipe, each colocated with a `.test.ts`; the barrel is a thin re-export and `"sideEffects": false` holds (no top-level registration).
