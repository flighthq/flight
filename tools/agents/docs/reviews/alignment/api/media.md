# API Alignment: @flighthq/media

**Verdict:** Naming, sentinels, `Readonly`, type imports, and audio/video symmetry are exemplary; the real gaps are an always-allocated `onComplete` signal (violating the opt-in signal-cost rule) and a missing teardown verb for channels that own WebAudio nodes and DOM event listeners.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| High | `playAudioResource`, `playVideoResource` (`onComplete: createSignal()`) | Each play unconditionally allocates a `Signal` (`AudioChannel.onComplete` / `VideoChannel.onComplete`). The map's signals rule states signal groups are opt-in via `enable*Signals` functions "which is when the associated cost is assumed." Here every channel pays for the signal whether or not anyone listens, and the cost is baked into the entity type rather than gated. | Move `onComplete` behind an `enableAudioChannelSignals(channel)` / `enableVideoChannelSignals(channel)` (returning the signal group, owned by this package, mirroring `enableStageSignals`/`enableNodeSignals`). Make the field nullable in `@flighthq/types` and emit only when enabled. |
| High | (missing) `disposeAudioChannel` / `disposeVideoChannel` | A channel keeps itself reachable through package-private runtime state: the audio runtime in the `channelRuntime` WeakMap (holding live `AudioBufferSourceNode`/`GainNode`) and, for video, an `'ended'` listener added to the `HTMLVideoElement`. The video listener is only detached inside `stopVideoChannel`/replay; there is no verb that fully detaches-and-releases a channel for GC. A caller who drops a channel without calling `stop` leaves the listener attached. | Add `disposeAudioChannel(channel)` / `disposeVideoChannel(channel)` that detach listeners, null the runtime nodes, and (once signals are opt-in) clear the signal group. `dispose*` is the correct verb here — these are detach-and-release-to-GC, not non-GC resource frees. |
| Low | `setAudioChannelGain`, `setAudioChannelCurrentTime`, `setAudioChannelPlaybackRate`, `setVideoChannelGain`, `setVideoChannelCurrentTime`, `setVideoChannelPlaybackRate` | All six `set*` mutators return the clamped/applied `number`. The map treats `set*`/`copy*` as in-place mutation; returning a value is a soft convenience that diverges from the void-mutator norm and does not map cleanly to the C/C++ free-function target (an out-param/void setter is the portable shape). Symmetric across audio/video, so at least internally consistent. | Decide one convention package-wide: either return `void` (caller reads back the field), or document the "returns the applied/clamped value" contract explicitly as a deliberate setter shape. Whichever is chosen should be applied consistently across sibling setters in the SDK. |
| Low | `setAudioChannelGain` / `setVideoChannelGain` (`gain`) | `gain` is the WebAudio term, but the user-facing playback vocabulary in OpenFL/Lime and most media APIs is `volume` (the video backend even assigns `element.volume = value`). The map favors the word "a reader would have reached for independently"; for a playback channel that is `volume`. Audio engineers say gain; app developers say volume. | Consider renaming the field/functions to `volume` (`setAudioChannelVolume`), or confirm `gain` is the intended canonical term given the broader audio-graph direction. Cross-package decision — surface rather than change unilaterally. |

## Clean

- **Full, unabbreviated type words** in every export: `getAudioChannelCurrentTime`, `setVideoChannelPlaybackRate`, `playAudioResource`, etc. No abbreviations of `AudioChannel`/`VideoChannel`/`AudioResource`/`VideoResource`.
- **Globally unique names** — every export is prefixed with its full entity (`*AudioChannel*` / `*VideoChannel*` / `*AudioResource` / `*VideoResource`); no bare `play`/`pause`/`stop` leaking from the barrel.
- **Sentinels for expected failure** — `playAudioResource` returns `null` when `source.buffer === null`; `playVideoResource` returns `null` when `source.element === null`. No throws anywhere; no validation of unreachable invariants.
- **`get*` is a real accessor** — `getAudioChannelCurrentTime` / `getVideoChannelCurrentTime` return computed `number` values, no boolean misuse.
- **`Readonly<T>` on options** — `Readonly<AudioPlayOptions>` / `Readonly<VideoPlayOptions>`; channel params are intentionally mutated by the setters, so non-`Readonly` is correct there.
- **`import type {}` on its own line** in both source files; all cross-package types (`AudioChannel`, `AudioPlayOptions`, `AudioResource`, and video equivalents) come from `@flighthq/types`, none defined inline (only the package-private `*ChannelRuntime` interfaces are local, which is correct).
- **Audio/video parameter and name symmetry** — the two channel families mirror each other 1:1 in function set, parameter order (`channel` first, `value` second), and return shapes.
- **Verb consistency for transport** — `play*`/`pause*`/`resume*`/`stop*` form a coherent, OpenFL-recognizable transport vocabulary applied identically across both channels; `play*Resource` (creates+starts from a resource) is a defensible domain verb and is symmetric.
