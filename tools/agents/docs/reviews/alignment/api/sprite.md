# API Alignment: @flighthq/sprite

**Verdict:** Strong overall — full type words, clean `create*`/`get*` quartets, `out`/`target` discipline, `Readonly<>` on inputs, and -1/no-op sentinels are all correct; the only real issues are an inconsistent `source` type on the `compute*LocalBoundsRectangle` family and an inconsistent mutating-entity parameter name (`target` vs `tilemap`).

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `computeQuadBatchLocalBoundsRectangle`, `computeParticleEmitterLocalBoundsRectangle` | Typed `source: Readonly<QuadBatch>` / `Readonly<ParticleEmitter>`, while `computeSpriteLocalBoundsRectangle` and `computeTilemapLocalBoundsRectangle` (and every `compute*LocalBoundsRectangle` in `@flighthq/displayobject` — `computeBitmapLocalBoundsRectangle`, `computeVideoLocalBoundsRectangle`, etc.) take `source: Readonly<Node>`. This is the `computeLocalBoundsRectangle` runtime-method contract, which is `(out, Node)`. Two members of the same family in one package disagree on the input type. | Type all four as `source: Readonly<Node>` and narrow internally (cast to `QuadBatch`/`ParticleEmitter` as the others already cast to `Sprite`/`Tilemap`), so the family is a uniform `(out: Rectangle, source: Readonly<Node>) => void`. |
| Medium | `reserveQuadBatch`, `resizeQuadBatch` vs `resizeTilemap`, `fillTilemapTiles`, `setTilemapTile` | The mutated entity is named `target` in the QuadBatch mutators but `tilemap` in the Tilemap mutators — same role, two names within one package. (Sprite/ParticleEmitter mutators add a third: `reserveParticleEmitter`/`setParticleEmitterLocalBoundsRectangle` use `target`.) | Pick one convention for the mutated-entity parameter across the package. The map's `out`/`target` guidance favors `target`; rename `tilemap` → `target` in `resizeTilemap`, `fillTilemapTiles`, `setTilemapTile`, `getTilemapTile` (or, if a domain-named first param is preferred for entity mutators, apply it everywhere — including `reserveQuadBatch`). |
| Low | `setQuadBatchLocalBoundsRectangle` | Uses `getDisplayObjectRuntime(target) as unknown as QuadBatchRuntime` (double cast) whereas the identical pattern in `setParticleEmitterLocalBoundsRectangle` uses a single `as ParticleEmitterRuntime`. Inconsistent and the `as unknown` is stronger than needed. | Drop `as unknown`; use `as QuadBatchRuntime` to match the sibling. |
| Low | `getQuadBatchTransformStride` | Public root-barrel export named with the abbreviated noun `Stride` and no `QuadBatch` qualifier (the other quad exports are `getQuadBatch*`). `Quad` alone is not a package entity (`QuadBatch` is); the name reads as operating on a `Quad`. Borderline for global uniqueness from the barrel. | Consider `getQuadBatchTransformStride` for symmetry with `getQuadBatchCapacity`/`getQuadBatchRuntime`, or confirm `QuadTransformType` (the type it operates on) justifies the `getQuadTransform*` prefix. |

## Clean

- **Full, unabbreviated type words everywhere.** `computeSpriteLocalBoundsRectangle`, `getParticleEmitterCapacity`, `setTilemapTile`, `hitTestQuadBatchPointXY` — no `getDOBounds`-style abbreviation; even `Rectangle`/`Point` are spelled in full.
- **`create*` quartet symmetry.** Each entity exposes the same shape: `create<Entity>`, `create<Entity>Data`, `create<Entity>Runtime`, `get<Entity>Runtime` across Sprite/QuadBatch/Tilemap/ParticleEmitter.
- **Allocation by verb.** `create*` allocate; `compute*LocalBoundsRectangle` and `copyLocalBoundsRectangle` write into the `out: Rectangle` and allocate nothing; `reserve*`/`resize*` mutate `data` typed arrays in place. Hot-path bounds/hit-test code is allocation-free.
- **`get*` vs `has*`/`is*`.** `get*` accessors all return values (runtime, capacity, stride, tile id); no boolean `get*` and no missing `has`/`is`. No misused `dispose*`/`destroy*` (none present, correctly — these entities own no non-GC resource here).
- **Sentinels for expected failure, no throws.** `getTilemapTile`, `hitTestQuadBatchPoint`, `hitTestQuadBatchPointXY` return `-1`; `setTilemapTile` no-ops on out-of-range; nothing throws for expected-missing cases.
- **`Readonly<T>` discipline.** Every read-only entity/point input is `Readonly<...>` (`Readonly<QuadBatch>`, `Readonly<Vector2Like>`, `Readonly<Rectangle>`); `get*Runtime` returns `Readonly<...Runtime>`; mutating params are deliberately non-`Readonly`. Primitives left bare, as the rule allows.
- **Alias-safe `out` functions.** `compute*LocalBoundsRectangle` read from `source.data` only and write to a distinct `out: Rectangle`; no read-after-write hazard.
- **`hitTestQuadBatchPoint` / `hitTestQuadBatchPointXY` pair** mirrors the `Point`/`PointXY` convention; the `Point` form delegates to the `XY` form (no duplicated logic).
- **Type imports.** All cross-package types come from `@flighthq/types` via dedicated `import type { ... }` lines (no inline `import { type Foo, bar }`, no inline cross-package type definitions). `*Kind` values imported as values.
