# API Alignment: @flighthq/velocity

**Verdict:** Mostly aligned — clean allocation/sentinel discipline, alias-safe `out` writer, correct `import type` and `@flighthq/types` sourcing; the notable gap is that the source-keyed accessors omit the operand type word from their names (`getVelocity` rather than the `*NodeVelocity` form the type docs themselves anticipate) and `source: object` parameters are not `Readonly<>`.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `contributeVelocity`, `getVelocity`, `hasVelocity`, `suppressVelocity`, `ensureVelocitySample` | The names omit the type word for what they key on. Each operates on a `VelocityField` keyed by a `source` object, but the name carries only the bare concept "Velocity". The convention ("globally self-identifying without context", full type word) wants the operand spelled out, as the scene-graph siblings do (`getNodeParent`, `containsNodeChild`). `getVelocity`/`hasVelocity` in particular are generic enough to collide conceptually from the root barrel. | Settle on the operand word and apply it consistently — e.g. `getSourceVelocity` / `hasSourceVelocity` / `contributeSourceVelocity` / `suppressSourceVelocity`, or the `*NodeVelocity` form the type doc already uses (see next row). |
| Medium | `contributeVelocity`, `suppressVelocity` | Doc-vs-impl drift: `@flighthq/types` `Velocity.ts` (the `VelocityContributor` doc comment, line 36–38) names the contract `contributeNodeVelocity` / `suppressNodeVelocity`, but the actual exports are `contributeVelocity` / `suppressVelocity`. The header layer and the implementation disagree on the canonical name. | Reconcile the two: rename the exports to match the documented `*NodeVelocity` names (and fix `getVelocity`/`hasVelocity` in step), or update the type doc to the chosen names. The header is the design surface, so the export names should follow whatever it settles on. |
| Low | `ensureVelocitySample` | The `ensure*` verb is outside the canonical allocation-verb set (`create*`/`clone*`/`acquire*` allocate; `get*` accesses). `ensureVelocitySample` is a get-or-create that may allocate a new `VelocitySample`, so its allocating behavior is not signalled by an approved verb. It also returns the internal mutable `VelocitySample` directly, exposing per-frame internals across the package boundary. | If the get-or-create must stay public, prefer a name whose verb is in the set (e.g. an `acquire*`-style or an explicit `get*`+`create*` split). Confirm exposing the mutable `VelocitySample` is intended public API; if it is an internal helper shared only with `contributeTransformVelocity`, it does not need to be in the root barrel. |
| Low | `contributeVelocity`, `getVelocity`, `hasVelocity`, `suppressVelocity`, `ensureVelocitySample`, `contributeTransformVelocity` | `source: object` / `root` parameters are never mutated by these functions but are typed as bare mutable `object` (and `contributeTransformVelocity` already uses `Readonly<Transform2DNode>` for `root`, so the convention is applied unevenly). The `Readonly<T>` rule wants read-only object params marked. | Type the never-mutated source params as `Readonly<object>` for consistency with `contributeTransformVelocity`'s `root`. |
| Info | `source: object` (all source-keyed functions) | The source is typed as the maximally-loose `object`. This is deliberate (the field is "source-agnostic — keys on any object, not only nodes"), but it gives the API no compile-time guard against passing a non-trackable value. | Acceptable given the intentional generality; flagged only for awareness. No change required unless a `VelocitySource` marker type is wanted. |

## Clean

- **Allocation by verb.** `createVelocityField` is the only root allocator and is correctly named; it returns the field via its constructor (object literal here is the constructor body, not a stray literal). `beginVelocityFrame`, `contributeVelocity`, and `suppressVelocity` mutate the field in place and allocate nothing.
- **Alias-safe `out` writer.** `getVelocity(field, source, out)` reads the sample's values and writes `out.x`/`out.y` from a distinct `sample.velocity` source; `out` aliasing an input is safe, and the function returns the `out` it filled — the correct `get*`-with-`out` shape. The zero-fill stale/missing path also writes `out` cleanly.
- **Sentinels, no spurious throws.** Expected-missing cases are handled with values, not exceptions: `getVelocity` zero-fills `out` for a missing or stale sample, `hasVelocity` returns `false`, and `ensureVelocitySample` creates rather than throwing. No function throws for an expected condition, and there is no validation of unreachable internal invariants.
- **Boolean predicate prefix.** `hasVelocity` correctly uses the `has*` prefix for its boolean return; `getVelocity` correctly returns a value (the filled `Velocity2D`), not a boolean.
- **Teardown verbs.** No teardown surface is misused — `suppressVelocity` is a per-frame zeroing operation (correctly _not_ named `dispose*`/`destroy*`), and the package owns no GC-rooting registrations or non-GC resources, so the absence of `dispose*`/`destroy*` is correct.
- **`Readonly<T>` where applied.** `contributeTransformVelocity`/`visitTransformVelocity` take `root`/`node` as `Readonly<Transform2DNode<Traits>>`, with the one deliberate `as Transform2DNode` cast commented as to why mutation is needed (world-transform ensure). The mutable `field` params are correctly mutable, since mutation is the intended effect.
- **Imports and type sourcing.** All cross-package types (`Velocity2D`, `VelocityField`, `VelocitySample`, `Transform2DNode`) come from `@flighthq/types`; none are defined inline. Every `import type {}` is on its own line, cleanly separated from the value imports in `transformVelocity.ts`.
- **`contributeTransformVelocity` naming.** The specialized transform-delta contributor correctly spells out its operand (`TransformVelocity`) and reads as the named baseline against the generic `contributeVelocity` — the one source-keyed export whose name is unambiguous.
- **Package shape.** Single `.` barrel export, `"sideEffects": false`, no top-level registration or mutable shared state; the field is caller-owned and passed explicitly. Dependencies (`geometry`, `node`, `types`) are minimal and `displayobject` is dev-only.
