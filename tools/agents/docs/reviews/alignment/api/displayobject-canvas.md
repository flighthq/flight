# API Alignment: @flighthq/displayobject-canvas

**Verdict:** Mostly aligned — strong, consistent backend-prefixed naming with full type words; the real issues are a few `undefined`-instead-of-`null` sentinels, the `release*` verb used for GC-only teardown, inconsistent `enable*` suffixing, and a handful of mutable params that should be `Readonly`.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `getCanvasCssFilter`, `getCanvasShapeCommand` | Return `string \| undefined` / `CanvasShapeCommand<any> \| undefined`. The SDK sentinel for an expected-missing lookup is `null`, not `undefined`. Especially jarring next to the sibling `resolveCanvasCssFilter`, which returns `string \| null` for the same concept (a missing CSS filter). | Return `... \| null` (`?? null` on the `WeakMap.get` / `Map.get`). Align `getCanvasCssFilter` with `resolveCanvasCssFilter`. |
| Medium | `releaseCanvasRenderCache` | Uses the `release*` verb, which is reserved for pool `acquire*`/`release*` brackets. There is no `acquireCanvasRenderCache`; its own comment says the target "holds no GPU handle; dropping the reference is enough to free it" — i.e. this is detach-to-GC, which is `dispose*`. (Note: `releaseGlRenderCache` / `releaseDomRenderCache` share the same drift, so this is a cross-backend naming decision, not a local slip — fix in concert.) | Rename to `disposeCanvasRenderCache` (and siblings), reserving `release*` for true pool brackets. |
| Medium | `enableCanvasRenderCache`, `enableCanvasTextInput` | Inconsistent `enable*` naming. The opt-in signal/feature group functions are `enableCanvasBlendModeSupport`, `enableCanvasClipSupport`, `enableCanvasCssFilterSupport` (all `…Support`), but these two drop the suffix. Same opt-in concept, two name shapes. | Pick one shape for opt-in feature installers. Either add the suffix (`enableCanvasRenderCacheSupport`, `enableCanvasTextInputSupport`) or drop it from the others; be consistent within the package and with sibling backends. |
| Low | `pushCanvasClipContours`, `pushCanvasClipRectangle` | `transform: Matrix` parameter is read-only in the body but typed mutable. Convention: `Readonly<T>` wherever mutation is not intended. | Type as `transform: Readonly<Matrix>` (matches `setCanvasTransform`, which already takes `Readonly<Matrix>`). |
| Low | `renderCanvasShapeCommands` | `commands: unknown[]` is only iterated/read but typed mutable. | Type as `commands: readonly unknown[]` (matches `buildScale9Mapper`/`mapCanvasScale9ShapeCommands`, which already use `readonly unknown[]` for the same command stream). |
| Low | `createCanvasRenderState` | `throw new Error('Failed to get context for canvas.')` on a failed `getContext('2d')`. A null 2D context is an environment/expected failure (context unavailable), not strictly a programmer-error precondition, so the throw-only rule is borderline here. | Acceptable as a misuse guard if a valid `<canvas>` is treated as a precondition; otherwise consider returning `null`. Low priority — document the precondition either way. |
| Low | `mapCanvasScale9ShapeCommands(out, source, mapper)` | Out-param placement is correct (`out` first), but the verb `map*` reads as a pure transform that allocates, while it actually writes into `out` in place. Minor: the name does not signal the out-param/in-place contract the way `copy*`/`set*` do. | Optional: the current name is defensible (it maps commands into `out`); no change required, noted for consistency review only. Alias-safety itself is handled correctly (guards `out !== source`). |

## Clean

- **Full, unabbreviated type words everywhere.** `drawCanvasDisplayObject`, `getCanvasRenderStateRuntime`, `ensureCanvasRenderCacheTarget`, `registerCanvasMaterialRenderer` — no abbreviated type names anywhere in the barrel.
- **Backend-prefixed, globally unique exports.** Every export is `*Canvas*`-prefixed, so names like `setCanvasTransform` / `drawCanvasShape` won't collide with the gl/dom siblings from the SDK barrel.
- **Allocation discipline by verb is correct.** `create*` (`createCanvasRenderState`, `createCanvasRenderTarget`, `createCanvasCacheState`) allocate; the hot-path writers (`setCanvasTransform`, `mapCanvasScale9ShapeCommands`, `pushCanvasClip*`) write into `out`/the context without returning fresh allocations.
- **Out-param alias-safety handled.** `mapCanvasScale9ShapeCommands` and its private `remapPathData` explicitly guard `out !== source` before copying, and read each cell before overwriting — safe when `out === source`.
- **`get*`/`is*`/`has*` discipline holds.** `get*` functions are all accessors; the boolean-returning helpers (`applyCanvasMaterial` returns whether it saved; `refreshCanvasRenderCache` returns whether a bake happened) are documented as status returns rather than mis-prefixed getters.
- **`Readonly<>` applied at the right seams.** `setCanvasTransform`/`beginCanvasRenderTarget` take `Readonly<Matrix>`; `pushCanvasClipRectangle`/`refreshCanvasRenderCache` take `Readonly<RectangleLike>` / `Readonly<RenderCacheRefreshOptions>`. Deliberate mutable returns (`createCanvasRenderStateRuntime`, `getCanvasRenderStateRuntime`) are commented as intentionally non-`Readonly` (per-frame writes).
- **`import type {}` hygiene.** Every file puts type-only imports on their own `import type { … }` lines; no inline `import { type Foo, bar }`. Cross-package types all come from `@flighthq/types` (`CanvasRenderState`, `RenderProxy2D`, `Scale9Mapper`, etc.) — none defined inline.
- **`enable*` opt-in / `register*` registration pattern** is used correctly for side-effect-free, tree-shakable feature install (no top-level registration), matching the SDK's renderer-registration rule.
