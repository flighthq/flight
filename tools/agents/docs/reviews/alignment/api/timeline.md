# API Alignment: @flighthq/timeline

**Verdict:** Strong, symmetric API — full type words, consistent verbs, clean `Readonly`/type-import hygiene; the one real issue is `getMovieClipSignals` doing lazy allocation under a `get*` name instead of following the sibling `enableNodeSignals`/`getNodeSignals` split.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| High | `getMovieClipSignals` | A `get*` accessor must not allocate or mutate, but this lazily creates and stores the signal group (`runtime.movieClipSignals ??= createMovieClipSignals()`). The sibling `@flighthq/node` codifies the opposite split: `enableNodeSignals` lazily creates, `getNodeSignals` is a pure accessor returning `NodeSignals \| null`. This package diverges from that established convention. | Rename the lazy-creating function to `enableMovieClipSignals` (matching the SDK `enable*` signal-group convention referenced in the CLAUDE map) and add a pure `getMovieClipSignals(clip): MovieClipSignals \| null` that returns the stored slot without creating it. |
| Low | `createTimeline` | Parameter typed `Partial<Timeline>` (bare), while every sibling constructor takes `Readonly<...>`: `createMovieClip(obj?: Readonly<PartialNode<MovieClip>>)`, `createMovieClipData(data?: Readonly<Partial<MovieClipData>>)`. The map requires `Readonly<T>` on object params where mutation isn't intended; `createTimeline` only reads `obj`. | Change signature to `createTimeline(obj?: Readonly<Partial<Timeline>>)`. |
| Low | `createTimelineSource` | Takes an inline anonymous object-literal param shape (`{ totalFrames?; frameRate?; labels?; constructFrame? }`) instead of a named `Readonly<...>` type. Cross-package shapes should live in `@flighthq/types`; an unnamed inline param is also not `Readonly`-wrapped and is awkward to reference. | Define a named `TimelineSourceInit` (or `Partial<TimelineSource>`-style) shape in `@flighthq/types` and accept `Readonly<TimelineSourceInit>`. |
| Low | `gotoAndPlayTimeline` / `gotoAndStopTimeline` (via `resolveFrame`) | Passing an unknown frame-label string throws `Error("Frame label ... not found")`. A missing label is an expected lookup failure, not programmer misuse — the package already models expected-missing with a sentinel (`findTimelineLabel` returns `null`). Throwing here is inconsistent with that. | Treat an unresolved label as a no-op (return without seeking) or clamp, mirroring how `gotoAnd*MovieClip` already no-ops when `timeline === null`. Reserve throwing for true precondition violations. |
| Info | `MovieClipSignals` (`onEnterFrame`/`onExitFrame`/`onFrameConstructed`) | The signal group is created but never emitted — `updateTimeline`/`fireConstructFrame` invoke `constructFrame` directly and never `emitSignal` on the group. Not an API-naming defect, but the exported `*Signals` surface is currently inert; flagged as a completeness gap. | Wire `updateTimeline`/the frame-construct path to emit the three signals when enabled, or remove the unused group until it is wired. |

## Clean

- **Full, unabbreviated type words everywhere.** Every exported name spells out `Timeline` or `MovieClip` in full — `getMovieClipCurrentFrame`, `gotoAndStopTimeline`, `setMovieClipSource`, `findTimelineLabel`. No abbreviations (`MC`, `TL`) anywhere.
- **Globally self-identifying, unique names.** Each function carries its operand type, so `playTimeline` vs `playMovieClip`, `updateTimeline` vs `updateMovieClip`, etc. read unambiguously from the barrel with no collisions.
- **Excellent verb symmetry between the two layers.** The `Timeline` engine and the `MovieClip` facade mirror each other 1:1: `play`/`stop`/`nextFrame`/`prevFrame`/`gotoAndPlay`/`gotoAndStop`/`update` exist as both `*Timeline` and `*MovieClip` with identical parameter order (`(entity, frame|deltaTime)`). This is exactly the cross-related-function symmetry the map asks for.
- **Allocation discipline by verb.** `create*` functions (`createTimeline`, `createMovieClip`, `createMovieClipData`, `createMovieClipRuntime`, `createMovieClipSignals`, `createTimelineSource`) are the only allocators; `update*`/`play*`/`goto*` mutate in place and allocate nothing in the hot path.
- **Correct accessor/boolean prefixes.** `get*` for value reads (`getMovieClipCurrentFrame`, `getMovieClipTotalFrames`, `getMovieClipRuntime`), `is*` for the boolean (`isMovieClipPlaying`) — no `get*` returning a boolean.
- **`Readonly` on getters and runtime access.** `getMovieClipRuntime(source: Readonly<MovieClip>): Readonly<MovieClipRuntime>`, `findTimelineLabel(timeline: Readonly<Timeline>, ...)`, and the internal helpers (`getTimelineFrameRate`, `getTimelineLabels`, `getTimelineTotalFrames`) all take `Readonly<Timeline>`.
- **Type-import hygiene.** Both files use a dedicated `import type { ... }` block with no inline `type` mixed into value imports; all cross-package types come from `@flighthq/types` (none defined inline). The `MovieClipKind` string-kind identity is imported from `@flighthq/types`, matching the kind-identity convention.
- **Sentinels for the common missing case.** `findTimelineLabel` returns `null`; the `*MovieClip` facade no-ops cleanly when `clip.data.timeline === null` rather than throwing.
- **Runtime-slot subsystem pattern followed.** Signals state is a nullable `movieClipSignals` slot on `MovieClipRuntime`, initialized to `null` in `createMovieClipRuntime` and lazily filled — consistent with the entity/runtime split (only the access-function naming, flagged above, deviates).
