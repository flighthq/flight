# API Alignment: @flighthq/displayobject-wgpu

**Verdict:** Largely conformant — full type words, correct `create*`/`destroy*`/`enable*`/`register*` verb discipline, and good `Readonly<>`/`out`-param hygiene — with a handful of real findings: an inline cross-package type, an `unknown[]`-typed seam that drops its type word, a `release*` that frees GPU resources, and an aliased-canvas re-export that risks signature drift.

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `drawWgpuColorTransformBitmap` | `renderProxy` is typed as an **inline anonymous structural object** (`{ alpha; transform2D: { a; b; c; d; tx; ty }; blendMode }`) defined in-package. Cross-package types must come from `@flighthq/types`, not inline. It is also the only `draw*` whose proxy param is **not `Readonly<>`** and not `RenderProxy2D`, breaking symmetry with the sibling `drawWgpuBitmap(state, renderProxy: RenderProxy2D)`. | Take `renderProxy: Readonly<RenderProxy2D>` (or a named `*Like` type exported from `@flighthq/types`) instead of the inline literal. |
| Medium | `remapWgpuScale9Commands(out: unknown[], source: readonly unknown[], mapper: Parameters<typeof mapCanvasScale9ShapeCommands>[2])` | The command stream is typed as `unknown[]` and the mapper type is pulled via `Parameters<typeof …>[2]` from the canvas package. The exported signature carries **no type word** for what it operates on and leaks an internal canvas function into the public type. The verb/`out`/`source` shape is correct; the typing is the issue. | Export a named command-list type (e.g. `CanvasShapeCommandList`) and a named mapper type from `@flighthq/types` and use them here, so the signature is self-describing and decoupled from `mapCanvasScale9ShapeCommands`'s positional params. |
| Low | `releaseWgpuRenderCache` | Body calls `destroyWgpuRenderTarget` — it **frees GPU textures** (a non-GC resource), which is `destroy*` semantics. The map reserves `release*` for pool/cache `acquire`/`release` brackets, and there is no `acquireWgpuRenderCache` partner (the lifecycle partners are `enableWgpuRenderCache` / `ensureWgpuRenderCacheTarget`). The verb implies "return to pool," not "free now." | Rename to `destroyWgpuRenderCacheTarget` (or `destroyWgpuRenderCache`) to match the `destroy*` = free-non-GC-resource-now rule, consistent with `destroyWgpuVideoData` / `destroyWgpuScale9ShapeData` in this same package. |
| Low | `registerWgpuShapeCommands` (re-export in `index.ts`) | Exported as an **alias of `registerCanvasShapeCommands`** from `@flighthq/displayobject-canvas`. Its public signature (`commands: CanvasShapeCommand<any>[]` — mutable array, `any` generic) is inherited from canvas and is not `Readonly<>`. The name reads as a wgpu-owned registration but is the canvas implementation; future canvas signature changes silently change this package's public API. | Acceptable as a deliberate shared-impl re-export, but make the intent explicit (the file already aliases a block of `defaultWgpu*` names this way). If wgpu shape-command registration is meant to be distinct, define a local wrapper typed against `@flighthq/types` rather than re-exporting the canvas symbol verbatim. |
| Low | `renderWgpuVelocity` | `root` and `target` are `Readonly<>` but `field: VelocityField` is mutable-by-default though it is consumed read-only (passed downstream, not mutated). Minor `Readonly<>` asymmetry within one signature. | Mark `field: Readonly<VelocityField>` to match the surrounding params, unless the velocity step deliberately mutates the field descriptor. |
| Info | `remapWgpuScale9Commands` | Uses a module-level scratch `_remappedPathData` written into `out[i+3]`; two `remapWgpuScale9Commands` results outstanding at once would alias the same backing array. Internal-only today (single call site drains immediately), but the shared scratch is not reflected in the `out`-param contract. | If this ever becomes reentrant, allocate per-call or document the single-result-at-a-time constraint. No change needed now. |

## Clean

- **Full, unabbreviated type words throughout.** Every exported name spells out its subject: `drawWgpuShape`, `drawWgpuTextLabel`, `createWgpuVideoData`, `registerWgpuColorTransformMaterials`, `pushWgpuClipRectangle`, `prepareWgpuSpriteBatchWrite`. `Scale9` is the canonical 9-slice term (`buildWgpuScale9Mapper`, `drawWgpuScale9Shape`), not an abbreviation. No `getDOBounds`-style shortenings.
- **Teardown verbs used correctly.** `destroyWgpuVideoData` / `destroyWgpuScale9ShapeData` / `destroyWgpuRichTextData` each free a GPU texture (`texture.destroy()`) — proper `destroy*` (free-non-GC-now) usage, distinct from a detach-to-GC `dispose*`. (The one drift is `releaseWgpuRenderCache`, flagged above.)
- **Allocation discipline by verb.** `create*` allocates (`createWgpuCacheState`, `createWgpuVelocityTarget`, `createWgpu*Data`); `ensure*` returns-or-lazily-creates a cached resource (`ensureWgpuQuadBatchResources`, `ensureWgpuRenderCacheTarget`); `draw*` hot-path functions write to GPU state without returning allocations.
- **`enable*` opt-in pattern for subsystems.** `enableWgpuClipSupport`, `enableWgpuRenderCache`, `enableWgpuTextInput` follow the map's opt-in `enable*` convention rather than registering at module top level (package is `"sideEffects": false`).
- **`register*` for backend/seam registration.** `registerDefaultWgpuMaterial`, `registerWgpuColorTransformShader`, `registerWgpuVelocityWriter`, `registerWgpuTextInputOverlay` — consistent registration verb.
- **`get*` accessors return values, not booleans.** `getWgpuRenderCacheTarget` (→ `WgpuRenderTarget | null`), `getWgpuVelocityWriter` (→ writer | null), `getWgpuQuadBatchPipeline`, `getWgpuQuadBatchPreludeWGSL` (→ string). No boolean `get*`; boolean-returning behavior is in `refreshWgpuRenderCache` (→ `boolean`), appropriately not named `get*`.
- **Sentinels for expected failure.** `buildWgpuScale9Mapper` returns `null` on degenerate input; `getWgpuRenderCacheTarget` / `getWgpuVelocityWriter` return `null` for misses. The single `throw` (`renderWgpuVelocity`, "call renderWgpuBackground first") is a genuine precondition/API-misuse guard, which is the sanctioned use of `throw`.
- **`Readonly<>` on input descriptors.** `pushWgpuClipRectangle(rect: Readonly<RectangleLike>, transform: Readonly<MatrixLike>)`, `pushWgpuClipContours(..., worldTransform: Readonly<Matrix>)`, `drawWgpuVelocityQuad(ctx: Readonly<WgpuVelocityContext>, …)`, `buildWgpuScale9Mapper(bounds: Readonly<RectangleLike>, scale9Grid: Readonly<RectangleLike>, …)`.
- **`out`/`source` parameter naming and order.** `remapWgpuScale9Commands(out, source, …)` puts the mutable target first with a `readonly` source — correct out-param convention (typing aside).
- **`import type {}` on its own line.** No mixed `import { type Foo, bar }` violations anywhere in the package.
- **Globally self-identifying `Wgpu` namespace.** Every exported name carries the `Wgpu` backend token, keeping root-barrel names unique against the `Canvas`/`Gl`/`Dom` siblings.
