# API Alignment: @flighthq/text

**Verdict:** Largely conformant — clean out-param/sentinel/`import type`/`Readonly` discipline; the notable gaps are the `set*String` setters that abbreviate the `text` field word, two `set*` setters that drift from their backing field name, and one asymmetric per-kind seam (`buildRichTextLayoutParams` exported while `buildTextLabelLayoutParams` is private).

## Findings

| Severity | Symbol | Issue | Suggested fix |
| --- | --- | --- | --- |
| Medium | `setTextLabelString`, `setRichTextString`, `setNativeTextString` | Setter is named `String` but the field it writes is `text` (`TextLabelData.text`, `RichTextData.text`, `NativeTextData.text`). Every sibling setter is field-faithful (`setTextLabelWidth`→`width`, `setNativeTextHeight`→`height`), so this breaks the `set<Type><Field>` symmetry and is not the actual type word being operated on — `String` is the value's primitive type, not the property. | Rename to `setTextLabelText` / `setRichTextText` / `setNativeTextText` to match the `text` field and the rest of the package's setter naming. (If the doubled `TextText` is judged awkward, that is a signal the field should be renamed — but the function must track the field, not vice versa.) |
| Medium | `setTextLabelFormat` | Sets the `textFormat` field but is named `Format`, dropping the field's `text` qualifier. Same field-name drift as the `*String` setters; inconsistent with field-faithful siblings. | Rename to `setTextLabelTextFormat` to match the `textFormat` field, or rename the field — keep function and field in lockstep. |
| Medium | `buildRichTextLayoutParams` | The per-kind layout-params seam is exported for RichText but its exact sibling `buildTextLabelLayoutParams` is a private function in `textLabel.ts`. Two implementations of the same seam with opposite visibility is asymmetric public surface; a consumer/agent sees one half of a pair. | Either export both (consistent seam) or keep both private and assign onto the runtime internally (consistent encapsulation). Decide one visibility for the seam. |
| Low | `RichTextDataInternal` (`internal.ts`) cast for `scrollH`/`scrollV` | Uses the `data as RichTextDataInternal` cast to write `scrollH`/`scrollV`, the `internal.ts` pattern the codebase map explicitly calls a legacy approach ("do not extend it; prefer runtime slots instead"). `setRichTextScrollH`/`setRichTextScrollV` and `createRichTextData` all route through it. | Move mutable scroll state to a narrow runtime slot, or make the fields normally writable in `@flighthq/types`, rather than casting away `Readonly` per call site. |
| Low | `dispatchRichTextWheel` | Named `dispatch*`, which elsewhere denotes input-event handlers driving a manager (`dispatchTextInput*`, `dispatchInteraction*`). This one only nudges `scrollV` on the field with no manager and no event-routing — closer to a `scroll*`/`setRichTextScroll*` mutation than an event dispatch. | Acceptable if treated as the field-level wheel-input entry point (it does mirror the `dispatch*Wheel` family). Consider `scrollRichTextByWheel` if it should read as a mutation rather than an event dispatch. Verb is consistent with the wheel-handler family, so low priority. |

## Clean

- **Full type words in names.** No abbreviated type words: `computeTextLabelLocalBoundsRectangle`, `getRichTextPasswordCharacter`, `createNativeTextRuntime` all spell out the entity. No `getDOBounds`-style contractions.
- **Allocation discipline by verb.** `create*` (`createTextLabel`, `createRichTextData`, `createNativeTextRuntime`) are the only allocators; `compute*LocalBoundsRectangle` and `getTextLayoutMetrics` write into an `out: Rectangle`/`out: TextMetrics` and allocate nothing. `build*LayoutParams` allocating a params object is consistent with the SDK-wide `build*` assembling verb (`buildSurface*ColorMatrix`, `buildTilesetRegions`).
- **out-param placement and alias-safety.** `out` is consistently first (`compute*(out, source)`, `getTextLayoutMetrics(out, source)`), matching geometry/bounds conventions. `out` (Rectangle/TextMetrics) and `source` (Node) are always distinct types, so no input/output aliasing is possible; each reads `data` fields before writing `out`.
- **Sentinels over throws.** No thrown errors for expected-missing cases. `getTextLayout` returns `null` when no measure provider is registered; `getRichTextPasswordCharacter` returns `null` when not masking; `getTextLayoutMetrics` zeroes `out` instead of throwing. No validation of unreachable internal invariants.
- **Accessor verbs.** `get*` used only for true accessors (`getTextLabelRuntime`, `getTextLayout`, `getRichTextPasswordCharacter`); none return a bare boolean, so no `has*`/`is*` is owed.
- **`Readonly<T>` on read-only references.** Source/query params are `Readonly<TextLabel>`, `Readonly<Node>`, `Readonly<TextLayoutResult>`; mutating setters correctly take a non-`Readonly` `source` because mutation is deliberate. `get*Runtime` return types are `Readonly<...Runtime>`.
- **`import type {}` on its own line.** All five files separate type imports into a dedicated `import type { ... } from '@flighthq/types'` block; value imports (`TextLabelKind`, `RichTextKind`, `NativeTextKind`) are split out. No inline `import { type Foo, bar }`.
- **Cross-package types from `@flighthq/types`.** No cross-package types defined inline; the only local type (`RichTextDataInternal` in `internal.ts`) is a derived `Omit<>` over the shared `RichTextData`, not a new cross-package contract.
- **Single root barrel.** `index.ts` is a thin `export *` re-export of the four source modules; no per-file `exports` subpaths, `sideEffects: false`.
- **Setter symmetry for dimension fields.** `set*Width`/`set*Height`/`set*AutoSize` are field-faithful and consistent across all three entities, each guarding with an equality early-return and invalidating bounds+content correctly.
