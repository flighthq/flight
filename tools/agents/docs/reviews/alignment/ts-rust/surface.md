# TS↔Rust Alignment: @flighthq/surface

**Verdict:** Function- and file-level alignment is excellent (all 85 native-core ops ported 1:1 with correct snake*case names and tracking filenames); the only real issue is an \_undocumented crate-name divergence in the sibling mixing crate* — TS `@flighthq/surface-rs` is implemented by the Rust crate `flighthq-surface-wasm` instead of the expected identity `flighthq-surface-rs`, which is what trips the structural gate.

## Name map findings

| TS symbol / file | Rust symbol / file | Issue |
| --- | --- | --- |
| `@flighthq/surface-rs` (package) | `flighthq-surface-wasm` (crate) | **Undocumented name divergence.** A real TS package `@flighthq/surface-rs` exists (the wasm-mixing shim). Package→crate is identity unless mapped, so the conformance script expects `flighthq-surface-rs` and reports `surface-rs` missing + `surface-wasm` unexpected (the 2 structural-gate violations). The crate even says `description = "...backs the @flighthq/surface-rs npm shim"`, confirming intent. Fix: rename the crate to `flighthq-surface-rs`, or add a `RENAMES` entry (`'surface-rs' → 'surface-wasm'`) in `scripts/rust-conformance.ts` + a divergence-map row with rationale. A rename is the cleaner choice since it restores identity. This is the surface _package's_ alignment story, so noting it here. |
| `createSurfaceFromCanvas` / `surfaceFrom.ts` | (none) | Expected. Browser-only (`getContext('2d')`); classified web-relocated in the conformance script and validated in `host-web`. Not a gap. The Rust crate keeps the substrate-independent siblings (`create_surface_from_image_source`, `create_surface_from_image_resource`) in `surface_from.rs`. |
| `encodeSurface` / `surfaceEncode.ts` | `encode_surface` / `encode.rs` | Present but a **documented-in-code stub**: returns empty `Vec` (no native PNG/JPEG codec chosen yet; TODO(wave-N)). Sentinel-correct, not silent drift, but behavior does not yet conform — worth a divergence-map note ("encode/decode pending native codec") so it is auditable rather than buried in a source TODO. |
| (none) | `decode_surface` / `encode.rs` | **Extra Rust function with no TS counterpart.** `@flighthq/surface` has no `decodeSurface` (decoding is done via resource/image loading on the web). Rust added `decode_surface` as the native mirror of `encode_surface` (also a stub returning `None`). Reasonable as the native seam, but it is an undocumented extra export — record it in the divergence map (Rust-only addition: native decode seam) or drop it until a codec lands. |
| `extractSurfacePixels32` / `surfacePixel.ts` | `extract_surface_pixels_32` / `pixel.rs` | In sync — digit gets its own underscore segment (`...pixels_32`), the conventional snake_case for a trailing numeral. Same for `writeSurfacePixels32` → `write_surface_pixels_32` and `rotateSurface180` → `rotate_surface_180`. Listed only to confirm these are aligned, not drift. |

## In sync

- **Crate name:** `@flighthq/surface` → `flighthq-surface` (identity) for the core ops crate itself.
- **All 85 native-core ops ported 1:1** with full unabbreviated type words and correct camelCase→snake_case: `applySurfaceColorTransform`/`applySurfacePaletteMap`/`applySurfaceThreshold`, the `build*ColorMatrix` family, `concatSurfaceColorMatrix`/`setSurfaceColorMatrixIdentity`, blur primitives (`blurSurfacePixelsHorizontal[Weighted]`/`...Vertical[Weighted]`, `boxBlurSurface`, `gaussianBlurSurface`, `computeGaussianKernel`, `buildSurfaceGradientRamp`), the surface-native op family (`bevelSurface`, `colorMatrixSurface`, `convolveSurface`, `displaceSurface`, `medianSurface`, `pixelateSurface`, `dropShadowSurface`, `glowSurface`, `innerGlowSurface`, `innerShadowSurface`, `sharpenSurface`, `gradientBevelSurface`, `gradientGlowSurface`), pixel access (`get/setSurfacePixel[Rgb]`, `getSurfacePixelChannel/Luminance`, `extract/writeSurfacePixels[32]`, `copySurfacePixels/Channel`, `convertSurfacePixelOrder`, `premultiply/unpremultiplySurfacePixels`), region/compose (`createSurfaceRegion`, `setSurfaceRegion`, `compositeSurfacePixels/Region`, `mergeSurface`, `drawSurface`), geometry (`resize/rotate[180/Clockwise/CounterClockwise]/flip[Horizontal/Vertical]Surface`, `scrollSurface`), fill/noise (`fillSurfaceRectangle/Noise/PerlinNoise`, `floodFillSurface`, `dissolveSurfacePixels`), morphology (`dilate/erodeSurface`), query/compare (`getSurfaceCoverage`, `getSurfaceColorBoundsRectangle`, `getSurfaceHistogram`, `equalizeSurfaceHistogram`, `getSurfaceMismatch`, `compareSurface[Fingerprints]`, `create/format/parseSurfaceFingerprint`), and `create/cloneSurface`, `create*FromImageSource/ImageResource`, `createImageResourceFromSurface`.
- **Filenames track cleanly.** Every TS `surface<Domain>.ts` maps to the same-basename Rust `<domain>.rs` (`surfaceBlur.ts`↔`blur.rs`, `surfaceColorMatrix.ts`↔`color_matrix.rs`, `surfacePaletteMap.ts`↔`palette_map.rs`, `surfaceFrom.ts`↔`surface_from.rs`, etc.). The Rust side correctly drops the redundant `surface` prefix the modular path already implies; `surface.ts`↔`surface.rs` and `index.ts`↔`lib.rs` as expected.
- **Conformance script reports 85/86 covered (1 gap = the web-relocated `createSurfaceFromCanvas`)** — the highest near-complete coverage of any nontrivial package, consistent with surface being a value-typed leaf and the prime mixing target.
- **Convention carry-through:** out-param ops take `&mut` outs; expected-failure paths return `Option` (`decode_surface`, `create_surface_from_image_resource`); the surface-op rename (dropping the `Filter` infix vs the `filters-surface` descriptor API) is already recorded in `conformance.md` (2026-06-23 re-layering entry).

### Divergence-map actions

1. **Resolve the `surface-rs`↔`surface-wasm` crate name** (rename crate to `flighthq-surface-rs`, or add a `RENAMES` entry + rationale). This is the only hard structural-gate failure for this package family.
2. Add a Rust-only note for `decode_surface` (native decode seam, mirrors `encode_surface`).
3. Optionally record that `encode_surface`/`decode_surface` are pending a native codec (currently sentinel stubs) so the behavior gap is auditable rather than source-TODO-only.
