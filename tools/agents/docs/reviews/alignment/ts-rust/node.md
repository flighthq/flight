# TS↔Rust Alignment: @flighthq/node

**Verdict:** Mostly aligned (68/74 TS exports map 1:1 with correct snake*case and full type words), but `flighthq-node` carries one **undocumented, conceptually-colliding rename** — the TS `Viewport` concept (`viewport.ts`, `createViewport`, `compute_viewport*_`) is renamed to `Scene`/`create*scene`/`compute_scene*_`in`scene.rs`, reusing the word `Scene`that the divergence map already assigns to the 3D`world → scene` rename — plus a large set of Rust-only arena accessors that are not recorded anywhere.

## In sync

- **Package→crate name** is identity (`@flighthq/node` → `flighthq-node`); correctly absent from the rename map.
- **68 of 74 TS exports port 1:1** with correct camelCase→snake*case and full unabbreviated type words: `add_node_child`, `add_node_child_at`, `compute_node_bounds_rectangle`, `compute_node_world_transform_revision`, `contains_node_child`, the four `convert_node_vector2/3*_`pairs,`create*node`, `create_node_runtime`, `create_node_signals`, `default_compute_local_bounds_rectangle`, `default_node_runtime_can_add_child`, `enable_node_signals`, all `ensure_node*_`(incl.`ensure*node_world_transform_matrix4`), all `get_node*_`revision/child/parent/root/runtime/signals/width/height/bounds-rectangle/transform-matrix accessors, all`init\_\_\_trait`(incl.`init_transform2d_trait`/`init_transform3d_trait`and the`\_runtime_trait`variants), all`invalidate_node_\*`, `remove_node_child`/`\_at`/`\_children`, `set_node_child_index`, `set_node_enabled`, `set_node_width`/`\_height`, `swap_node_children`/`\_at`.
- **Type-word preservation is clean** on the matched set — `Vector2`/`Vector3`/`Matrix4`/`Transform2D`/`Transform3D` digits stay attached (`convert_node_vector2_global_to_local`, `ensure_node_world_transform_matrix4`), never abbreviated.
- **Filenames track** for `boundsRectangle.ts↔bounds_rectangle.rs`, `hierarchy.ts↔hierarchy.rs`, `node.ts↔node.rs`, `revision.ts↔revision.rs`, `transform2d.ts↔transform2d.rs`, `transform3d.ts↔transform3d.rs`, `hasAppearance.ts↔appearance.rs` (the `has`-prefix drop is consistent with Rust's trait modeling), and `index.ts↔lib.rs` (conventional barrel rename).
- **Convention carry-over** is correct where it applies: out-params → `&mut`/`out`, sentinels → `Option`/`-1` (e.g. `get_node_child_at` → `Option`, `get_node_child_index` → `-1`). No `dispose_*`/`destroy_*`/`acquire_*`/`release_*` apply to this tier in either language.
- The arena foundation (`NodeArena`, `NodeId`, `KindId`) is the documented Rust-port decision, correctly local to the crate and re-exported.

## Suggested divergence-map additions

The map currently records **nothing** for `flighthq-node`, yet the crate has the single largest naming divergence in the audited set. Two entries are needed:

1. **`Viewport` → `Scene` (inside `flighthq-node`/`flighthq-types`).** Record that the TS `Viewport` concept (`createViewport`, `compute_viewport_*`, `ViewportAlign`, `ViewportScaleMode`) is renamed to `Scene`/`create_scene`/`compute_scene_*`/`SceneAlign`/`SceneScaleMode` in Rust — **with a rationale**, because this collides with the existing `world → scene` (line 37/134) 3D rename. As written, the divergence map's `scene` row implies one meaning; a reader cannot discover that `flighthq-node` _also_ owns a `Scene`. Strongly consider whether this rename should exist at all: keeping the Rust name `viewport`/`create_viewport` would both preserve 1:1 conformance and avoid the `Scene` overload. If the rename stays, it must be an explicit, rationalized map entry.

2. **`flighthq-node` Rust-only arena surface.** A note covering the `spatial2d_*` concrete-tier family, the `propagate_node_transforms*` arena walks, the revision/dirty plumbing (`is_dirty`, `next_revision`, `is_node_*_dirty`), and the convenience/alias accessors — all consequences of the slotmap-arena model. Within that note, flag two cleanups that are _not_ pure arena accommodations: (a) the display-object-tier `alpha`/`blend_mode`/`visible` accessors that widen the node tier beyond its TS scope, and (b) the duplicate `get_bounds_node_*`/`invalidate_bounds_node_*` names that coexist with the ported `get_node_*`/`invalidate_node_*` ones — pick one tier word.

No existing map entry looks stale for this crate, because none exists yet.
