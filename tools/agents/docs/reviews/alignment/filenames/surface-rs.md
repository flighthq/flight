# Filename Alignment: @flighthq/surface-rs

**Verdict:** Not a render backend-variant package (`-rs` is the wasm "mixing" drop-in for `@flighthq/surface`, not one of the `-canvas`/`-dom`/`-gl`/`-wgpu` render backends), so the prefix-first backend-token rule does not apply in its strict form — yet the package still embeds a backend distinction (wasm vs. the JS `@flighthq/surface`), and its committed source names it correctly with a `Wasm` token. Every hand-authored file names the domain (`surface`) plus the wasm backend it crosses into, every test mirrors its source, and the only snake_case name is a generated wasm-pack artifact whose name Flight does not own. No defects.

## Findings

| File | Issue | Suggested rename |

| `src/wasm/surface_wasm_bg.wasm` | Generated binary artifact emitted by `wasm-pack` (`--out-name surface_wasm`), not hand-authored Flight source. snake_case + `_bg` suffix are wasm-pack's fixed output convention, not a Flight naming choice; the package map's filename rules govern authored `.ts` source, not the tool-emitted `.wasm` blob. Flagged only for completeness — it is the one name that breaks the camelCase norm, but renaming it is not Flight's to make (it is the wasm-pack contract the glue imports). | None — leave as wasm-pack emits it. ## Clean

These pass the "remove the folder" test — each bare filename names the surface domain plus the wasm backend seam (the legitimate backend distinction for this mixing package), and tests mirror their source:

- `index.ts` — package barrel; thin re-export shim (overrides the bulk pixel ops with the wasm versions, then `export *` from `@flighthq/surface`). Barrel name is correct; not a dumping ground.
- `surfaceWasm.ts` — names the domain (`surface`) and the backend it marshals into (`Wasm`). Self-identifying as "the wasm-backed surface pixel ops," which is exactly this file's role. The wasm token here is the analogue of the render packages' prefix-first backend token; because `surface-rs` is a single-backend mixing package rather than a render backend-variant, a domain+`Wasm` name (not a prefix-first `wasmSurface.ts`) reads correctly and matches the JS counterpart's `surface*` ordering it must shadow.
- `surfaceWasm.test.ts` — colocated, mirrors `surfaceWasm.ts` exactly (and the test asserts wasm output against the JS `@flighthq/surface` reference — the conformance posture for the mixing seam).
- `src/wasm/surfaceWasmBytes.ts` — names the object it holds: the base64-embedded wasm module bytes. Generated by `scripts/embed-wasm.ts`, header-marked GENERATED, self-describing at a glance.
