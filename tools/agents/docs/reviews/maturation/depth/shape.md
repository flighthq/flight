# Maturation Roadmap: @flighthq/shape

**Current verdict:** solid — 78/100; a faithful, near-complete port of the Flash/OpenFL `Graphics` drawing surface that falls short of authoritative on a handful of missing canonical commands, an inert (handler-less) hit-test registry, dropped winding, control-hull cubic bounds, and no consumer-facing demonstration.

The present surface is a strong base: the full `appendShape*` command vocabulary (pen path, primitives, four fill kinds, the complete stroke-style signature, raw `drawPath` injection), the entity/runtime quartet (`createShape`/`createShapeData`/`createShapeRuntime`/`getShapeRuntime`, `ShapeKind`), a real scale-9 quartet (`createScale9Shape` …, `Scale9ShapeKind`), exact-quadratic-extrema `computeShapeLocalBoundsRectangle`, GPU solid-fill extraction (`getShapeFillRegions`/`hasNonSolidShapeFill`), and an extensible string-keyed hit-test registry (`registerShapeHitTestCommand`/`hitTestShapeCommandPoint`). Types live header-first in `@flighthq/types/ShapeCommand.ts` (style enums + the declaration-mergeable `ShapeCommandRegistry`). The Rust crate `flighthq-shape` already mirrors every source file (`shape.rs`, `shape_commands.rs`, `shape_fill.rs`, `scale9_shape.rs`, `shape_hit_test_registry.rs`, `command_buffer.rs`), so every tier below carries a parallel Rust obligation.

Because the package is already solid, Bronze is the close-the-gaps work the depth review names as priorities, Silver is professional completeness (introspection, ergonomics, consistency), and Gold is the genuine authoritative frontier. Every tier is cumulative and overwhelmingly additive — none of it conflicts with the plain-data / free-function / explicit-allocation / tree-shakable style.

## Bronze

The smallest set that closes the depth review's correctness and "exhaustive `Graphics`" gaps — the 20% that makes the package genuinely complete for its stated domain. Each is dependency-light and individually testable.

- **`appendShapeDrawTriangles`** — the headline missing command. `appendShapeDrawTriangles(shape, vertices, indices, uvtData?, culling?)` pushing a `drawTriangles` entry. Add the `drawTriangles` key to `ShapeCommandRegistry` in `@flighthq/types` first (tuple: `vertices: number[]`, `indices: number[] | null`, `uvtData: number[] | null`, `culling: TriangleCulling`), plus a `TriangleCulling` type (`'none' | 'positive' | 'negative'`). Teach `computeShapeLocalBoundsRectangle` to bound the vertex set, and `getShapeFillRegions`/`hasNonSolidShapeFill` to treat textured (uvtData-bearing) triangles as non-solid.
- **Exact cubic-bezier extrema in `computeShapeLocalBoundsRectangle`.** Replace the control-hull fallback for `cubicCurveTo` (and `drawPath` `CUBIC_CURVE_TO`) with derivative-root solving (the quadratic-in-t roots of the cubic derivative per axis), matching the existing quadratic-extrema treatment. Pure correctness fix; tightens over-reported bounds. Mirror in `shape.rs`.
- **Honor `drawPath` winding in `getShapeFillRegions`.** Stop hardcoding `winding: 'nonZero'`; carry the per-`drawPath` `PathWinding` into the emitted `ShapeFillRegion.path.winding`, and default primitive/pen-path fills to a documented winding (nonZero) explicitly rather than silently. Correctness fix the depth review flags.
- **`getShapeGraphicsData` (typed readback).** A typed decoder over the flat `[key, argCount, ...args]` buffer returning `readonly ShapeGraphicsRecord[]` (a discriminated union keyed by `ShapeCommandKey`, defined in `@flighthq/types` over the existing `ShapeCommandRegistry`). The `readGraphicsData` round-trip the depth review names — the first public typed view of the buffer, foundation for inspection, serialization, and tooling. Pair with **`forEachShapeCommand(shape, visitor)`** for an allocation-free walk.
- **Default built-in hit-test handlers + `enableShapeHitTesting()`.** Ship handlers for the primitives (`drawCircle`, `drawEllipse`, `drawRectangle`, `drawRoundRectangle`) and a fill-region point-in-polygon for pen paths / `drawPath`, registered by an opt-in `enableShapeHitTesting()` (the `enable*` group pattern) so `hitTestShapeCommandPoint` is no longer inert in isolation. Keeps default handlers tree-shakable behind the opt-in.
- **`isShapeEmpty(shape)` and `getShapeCommandCount(shape)`.** The small emptiness/count ergonomics a retained-command API is expected to expose; both are trivial and high-traffic.
- **Naming + signature polish.** Rename `appendShapeRoundRectanglePath` → `appendShapeRoundRectangleVarying` (it expands four independent corner radii inline, not a raw path — the `Path` suffix misleads). Add `Readonly<Shape>` overloads to `getShapeFillRegions` / `hasNonSolidShapeFill` so callers stop passing the untyped `readonly unknown[]` buffer directly.

Effort: small–medium. `appendShapeDrawTriangles` and the typed readback are the two larger items (both need a `@flighthq/types` addition first); cubic bounds, winding, the count/empty queries, and the rename are small. All have direct Rust mirrors.

## Silver

Competitive and solid — what a well-regarded retained vector-shape / `Graphics` library offers for common professional use: full introspection/serialization, the remaining canonical commands, consistency across the fill/bounds/hit-test passes, and a demonstrable consumer story.

- **`appendShapeGraphicsData(shape, records)` — typed write/replay.** The inverse of `getShapeGraphicsData`: append a `readonly ShapeGraphicsRecord[]` onto the buffer (Flash's `drawGraphicsData`). Together with the Bronze readback this is a complete typed round-trip over the command stream — the canonical inspect-edit-replay surface.
- **Serialization helpers — `@flighthq/shape-formats` neighbor package.** `serializeShapeCommands(shape): ShapeCommandJson` and `parseShapeCommands(json): ShapeData` for a stable, versioned JSON form of the command stream (the "-formats" importer/parser pattern, off the core bundle since it carries no rendering weight). Decide the on-disk schema once in `@flighthq/types`; it doubles as the scene-migration form for shapes.
- **Curve-flatten consistency.** Have `computeShapeLocalBoundsRectangle` and the default `drawPath`/fill-region point-in-polygon hit handlers flatten curves with the same tolerance policy `getShapeFillRegions` hands the renderer, so bounds, hit testing, and fills agree. Surface a shared `flattenShapeCommands`-style internal (or delegate to `@flighthq/path`) rather than three independent walks.
- **Convenience primitives.** `appendShapeArc(shape, cx, cy, radius, startAngle, endAngle, anticlockwise)` and SVG-style `appendShapeArcTo(shape, ...)` (cubic-expanded on append so the buffer stays in the existing verb vocabulary), plus `appendShapePolygon(shape, points)` / `appendShapePolyline`. Flash lacks these, but a canonical vector-shape author expects them; circle/ellipse currently cover only the common cases.
- **Stroke-aware bounds.** Today `computeShapeLocalBoundsRectangle` expands by the _last_ `lineStyle` thickness globally; track the active thickness per fill/stroke span and account for join/miter overshoot (miter-limit-bounded) so stroked-shape bounds match the rendered extent. The right grade of correctness for "competitive."
- **Hit-test fill/stroke fidelity.** Extend the default handlers to respect fill winding (even-odd vs nonZero) and to optionally hit-test stroke outlines (within half-thickness of a segment), not just fills — matching what `@flighthq/interaction`'s `defaultShapeHitTestPoint` needs, so the in-package handlers are a true superset.
- **Bounds query convenience.** `getShapeBounds(shape, out)` as the public allocation-explicit wrapper over the runtime's `computeLocalBoundsRectangle` (callers currently reach through the runtime); `isShapeBoundsContainingPoint(shape, x, y)`.
- **Consumer-facing example + functional test.** A `tests/functional/shape-*` scene exercising every fill kind, strokes, scale-9, and the hit-test registry across Canvas/DOM/WebGL — the depth review's "no consumer-facing demonstration of the registries it exposes" gap. Validates cross-backend consistency of the fill-region GPU path vs. the raster fallback.

Effort: medium. The `-formats` neighbor package and the flatten-consistency refactor are the heavier items; arcs/polygons, stroke-aware bounds, and the convenience wrappers are small–medium. Pulls `@flighthq/path` harder for shared flattening. Surface the JSON schema and the neighbor-package split as design decisions before building.

## Gold

Authoritative / AAA — the canonical reference for a retained vector-shape display node. Exhaustive `Graphics` coverage, performance, full edge-case handling, complete tests/docs, and 1:1 Rust parity.

- **Complete `Graphics` command parity.** Audit against the full Flash/OpenFL `Graphics` surface and close every remaining command: `appendShapeBeginShaderFill` (over `@flighthq/materials`' shader seam), `appendShapeDrawTrianglesWithGradient`, `appendShapeDrawQuads` / `appendShapeDrawTiles` if not delegated to `@flighthq/sprite`, `lineShaderStyle`, and explicit fill-level winding control (`appendShapeBeginFillWithWinding`). Decide per command whether it belongs here or one layer over (raise cross-package overlaps with `@flighthq/sprite` / `@flighthq/materials`).
- **Typed-array / pooled command buffer.** Offer a `Float32Array`-backed numeric buffer variant (with the object-bearing entries — bitmaps, matrices, color/ratio arrays — in a parallel side table) for the per-frame rebuild hot loop, plus `acquireShapeFillRegions` / `releaseShapeFillRegions` pooling so `getShapeFillRegions` is allocation-free in steady state. Benchmark suite gating regressions.
- **Robustness & edge cases.** Degenerate primitives (zero/negative radius, zero-size rect, NaN/Infinity inputs) returning sentinels or no-op per the throw-only-on-misuse rule; self-intersecting fill paths and overlapping `drawTriangles` documented and handled by the same robustness policy as `@flighthq/path`; empty-fill / unmatched `endFill` / mid-stream `clear` handled deterministically. Explicit epsilon policy shared with `@flighthq/path`.
- **Exhaustive measurement.** `getShapeContourCount`, per-fill-region bounds, `getShapeNearestPoint` (closest point on outline), and tight bounds that account for gradient-fill matrices and bitmap-fill tiling extents (not just geometry) where they affect the rendered box.
- **Scale-9 completeness.** Distortion-correct command rewriting under a `scale9Grid` (the canonical nine-slice that stretches edges/center and pins corners), `computeScale9ShapeLocalBoundsRectangle`, and a functional test proving corner/edge/center behavior across backends — today the quartet carries the grid but the stretch semantics live in the renderer; make the in-package contract authoritative.
- **Signals (only if a live use emerges).** No mutable-shape-observation case exists today. If an editing/authoring tool ever needs change notification on the command stream, gate it behind an `enableShapeSignals()` group in this package — do not add speculatively.
- **Full test & doc coverage.** One colocated `*.test.ts` per source (already the pattern), `out`-aliasing tests on `computeShapeLocalBoundsRectangle` and any new out-param functions, the parser/serializer round-trip corpus, and an `npm run api shape` surface that reads as a complete, symmetric vocabulary.
- **1:1 Rust parity.** Every Gold function ported to `flighthq-shape` with conformance-tested output. The command stream is plain data, so the typed readback/write, bounds, fill-region extraction, and hit testing are deterministic and headlessly fingerprint-able — prime conformance targets. Triangles, cubic-extrema bounds, scale-9 rewriting, and the JSON form must match TS bit-for-bit on the conformance corpus.

Effort: large but independently shippable per item. The typed-array/pooled buffer and exhaustive scale-9 are the heaviest; the remaining commands and robustness work are medium. Boolean/clip-style fill resolution is explicitly out of scope (it belongs to `@flighthq/path`).

## Sequencing & effort

Recommended order, with dependencies and cross-package / design-decision items called out.

1. **Header-layer decisions first (block the Bronze code).** In `@flighthq/types/ShapeCommand.ts`: (a) add the `drawTriangles` key + `TriangleCulling` type; (b) define the `ShapeGraphicsRecord` discriminated union over the existing `ShapeCommandRegistry` (this is the typed view both readback and write depend on, plus the Rust mirror). Settle both before writing Bronze.
2. **Bronze build order:** cubic-extrema bounds fix → honor `drawPath` winding in `getShapeFillRegions` → `isShapeEmpty`/`getShapeCommandCount`/`forEachShapeCommand` → `getShapeGraphicsData` → `appendShapeDrawTriangles` (with bounds + fill-region handling) → default hit-test handlers + `enableShapeHitTesting()` → naming/signature polish (`appendShapeRoundRectangleVarying` rename, `Readonly<Shape>` overloads). The two correctness fixes are smallest and unblock nothing else, so they land first; the rename is a breaking change but the pre-release/no-consumers status makes that free — do it now, not later.
3. **Dependency note.** Shared curve flattening (Silver) and any delegation of primitives/arcs likely add or deepen the dependency on `@flighthq/path`; the package currently depends on `@flighthq/displayobject`, `@flighthq/geometry`, `@flighthq/node`, `@flighthq/types`. Add `@flighthq/path` to `package.json` + `tsconfig.json` references when that work begins, then run `npm run packages:check`.
4. **Silver build order:** typed write `appendShapeGraphicsData` (pairs with Bronze readback) → flatten-consistency refactor (shared internal or `@flighthq/path` delegation) → arcs/polygons → stroke-aware bounds → hit-test fill/stroke fidelity → `getShapeBounds` wrapper → the functional-test/example scene last (it validates everything above across backends).
5. **Cross-package / new-package decisions to surface (do not take autonomously):** (a) `@flighthq/shape-formats` for serialize/parse — copy a nearby package shape, run `npm run packages:check`, raise with the user; (b) the JSON/scene-migration schema lives in `@flighthq/types` and round-trips with scene versioning — coordinate with the types-layout owner; (c) primitive/arc overlap with `@flighthq/path`'s `appendPath*` builders — decide whether shape delegates down rather than forking two implementations (the path roadmap flags the same overlap; resolve it once for both); (d) Gold shader/quad/tile commands overlap with `@flighthq/materials` and `@flighthq/sprite` — settle ownership before adding.
6. **Renderer coordination.** Honoring `drawPath` winding (Bronze) and stretch-correct scale-9 (Gold) change what the renderer packages (`displayobject-canvas`/`-gl`/`-wgpu`) consume from the fill-region / command path — coordinate with those owners and gate with `npm run test:functional:parity` so the GPU fill route and raster fallback stay consistent.
7. **Rust parity runs alongside each tier, not after.** `flighthq-shape` mirrors the same files today; keep it in lockstep so the conformance corpus grows tier-by-tier. The deterministic, plain-data functions (typed readback/write, cubic-extrema bounds, fill-region extraction, hit testing, JSON form) are the highest-value early conformance targets — fingerprint-test them as each lands.
